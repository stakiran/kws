# (添付1) (添付2) fable 5とopus 4.8のシステムプロンプトです。違いを論じて

大きく言うと、**Fable 5のプロンプトは「超高性能モデルを、安全・長文・知的作業向けに慎重運用する設計」**、 **Opus 4.8のプロンプトは「現行最高公開モデルを、検索・ツール・アプリ連携込みで実用運用する設計」** に見えます。
ただし、ここでは添付テキストの内容を前提にした比較で、実際の公式プロンプトかどうかは検証していません。

## 1. モデル階層の語り方が違う

Fable 5側は、Claude Fable 5を「Claude 5ファミリー」「Mythos-class」「Opusより上位」と位置づけ、Fable 5とMythos 5が同じ基盤モデルで、Fable 5には追加のデュアルユース安全措置がある、というかなり強い“次世代フロンティア”表現になっています。APIモデル一覧にも `claude-fable-5` が入り、Fable 5が一般利用可能な最上位モデルとして扱われています。

一方Opus 4.8側は、Claude Opus 4.8を「最新のClaudeモデル」「公開されている最先端」と位置づけつつ、Mythos Previewは非公開で、Project Glasswingの信頼組織向け、という構図です。つまりOpus 4.8は“公開最上位”、Fable 5は“公開最上位をさらに更新した新世代”として書かれている。

ここは思想差というより、**プロダクト世界線の差**が大きいです。Fable側は「Opusの上にMythos/Fable層が来た」未来設定、Opus側は「Opus 4.8が公開最上位で、Mythosは非公開プレビュー」という設定です。

## 2. Opus 4.8は「検索ファースト」が明示的で強い

Opus 4.8側には `<search_first>` があり、「現在の世界に関する factual question は必ず検索してから答える」とかなり直球で命令されています。価格、現職者、法律、新しいカテゴリなど、モデルが知っていそうでも検索せよ、という運用思想です。

Fable 5側にも知識カットオフ後や現在性のある事柄では検索する指示がありますが、検索の位置づけは「必要時に最新情報を取る」寄りです。さらに検索利用ガイドラインや著作権ルールがかなり細かく入っており、検索そのものよりも**検索結果をどう安全・高品質・非侵害に使うか**へ比重があるように見えます。 

なので、検索運用では、Opus 4.8は「まず検索」、Fable 5は「検索も含めた高精度リサーチ作法」という感じです。Opusは行動規則が太く、Fableは品質管理規則が厚い。

## 3. 安全方針は共通だが、Opus 4.8は武器・CBRN周りがより明文化されている

両方とも児童安全、危険物、マルウェア、自傷、摂食障害、政治的中立性などの大枠はかなり共通しています。Fable 5側は児童安全で「パターンレベルに留める」「拒否時に検出メカニクスを語らない」といった、悪用可能な境界説明を避ける細かい配慮が見えます。

Opus 4.8側は、武器・CBRN・通常兵器まで含めて「意味ある uplift を与えるか」を基準にし、会話全体の累積出力を見て止める、という記述がかなり強いです。これは単発拒否ではなく、**セッション全体で危険な設計パッケージ化を防ぐ**思想です。

面白いのは、Opus 4.8には `<default_stance>` として「基本は助ける。具体的で深刻な害があるときだけ断る」というヘルプ志向が明示されている点です。
つまりOpus 4.8は「助けることをデフォルトにしつつ、危険領域では累積リスクを強く見る」。Fable 5は「高性能モデルだからこそ、児童安全・ウェルビーイング・著作権・デュアルユースに細かいガードを置く」という印象です。

## 4. ツール観はOpus 4.8のほうが“エージェント製品”寄り

Fable 5側もClaude Code、Cowork、Chrome/Excel/PowerPointなどを挙げていますが、Opus 4.8側はさらにClaude DesignやMCP registry、アプリ推薦、コネクタ探索のようなツール記述が見えます。特にOpus 4.8には「目に見えるツール一覧は一部で、必要なら tool_search で能力を探す」というメタなツール発見思想があります。

これはかなり重要で、Opus 4.8のプロンプトはチャットモデルというより、**接続先・アプリ・MCP・UIウィジェットを横断する操作主体**としてClaudeを設計している感じがあります。Fable 5はモデル能力と安全設計の説明が前面、Opus 4.8は製品内エージェントとしての接続性が前面、という差です。

## 5. 口調・フォーマットはOpus 4.8のほうが制約が細かい

両方とも「温かく、過剰な箇条書きを避ける」方向ですが、Opus 4.8側は「簡潔」「絵文字を使わない」「pet nameを使わない」「genuinely / honestly / actually を避ける」など、語彙レベルの制御が増えています。

Fable 5側も温かさ、成人として扱うこと、箇条書き抑制などはありますが、Opus側ほど“文体の癖を消す”細部指定は目立ちません。

ここから読むと、Opus 4.8は「ユーザーを圧倒しない、簡潔で製品的な応答」をかなり意識している。Fable 5は「高度な説明・長文・思考支援にも耐えるが、過剰フォーマットは避ける」寄りです。

## 6. ウェルビーイングはFable 5のほうが心理的推測への警戒が濃い

Fable 5側は、ユーザーが明示していない診断名を付けない、摂食障害に関する因果ストーリーを勝手に与えない、危機サービスの悪い経験を一般化しない、といった心理・医療コミュニケーション上の細かい禁止が多いです。

Opus 4.8側にも同種のウェルビーイング方針はありますが、Fable 5ほど「心理的解釈を与えすぎない」ことへの文章量は多くない印象です。

これはFable 5のほうが、より“賢いモデルがそれっぽい心理分析をしてしまうリスク”を見越しているように読めます。高性能化すると、危ないのは露骨な有害回答だけでなく、もっともらしい過剰解釈なんですよね。ここ、地味だけど大事です。

## 7. 記憶システムはかなり共通

両方に memory system があり、過去会話由来の記憶を自然に使うが、メモリシステムを不必要に語らない、センシティブ情報を不用意に持ち出さない、過度な親密さを演出しない、という方針は共通しています。

この部分はモデル差というより、Claude製品全体の統一設計に近い。違いをあえて言うなら、Fable 5側は長文会話・高能力モデル向けに、記憶の扱いと依存防止の注意が全体のウェルビーイング方針と結びついているように見えます。

## 総評

この2つのシステムプロンプトの差は、単に「Fable 5のほうが新しい／Opus 4.8のほうが古い」というより、**想定しているClaude像の違い**として読むと面白いです。

Fable 5は、より高能力な“知的パートナー”を危険にしないためのプロンプトです。高性能・長文・安全・著作権・心理的配慮・デュアルユース対策が厚い。特に「賢すぎるモデルが危ない助言や過剰解釈をしてしまう」ことへの予防線が強い。

Opus 4.8は、より“製品内エージェント”としてのClaudeを安定運用するプロンプトです。検索ファースト、ツール発見、Claudeアプリ群、MCP、UI、簡潔な文体、具体的な拒否境界が目立つ。チャット単体ではなく、外部世界と接続して動くClaudeとして整えられている。

一言で言えば、**Fable 5は「フロンティア能力を安全に封じ込めながら使わせるプロンプト」**、**Opus 4.8は「公開プロダクトとして、検索・ツール込みで確実に仕事をさせるプロンプト」**です。
Fableは研究所っぽい緊張感、Opusは製品チームっぽい運用感。かなりキャラが違います。
