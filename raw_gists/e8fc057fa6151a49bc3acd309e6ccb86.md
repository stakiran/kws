# Quartz・MkDocs・代替 SSG に関する Q&A まとめ

---

## Q1. Quartz とは何？

**A.** Markdown ファイル群を「デジタルガーデン」スタイルの静的サイトに変換する SSG。

- 入力: `content/` フォルダ配下の `.md` ファイル
- 出力: `public/` に静的な HTML/CSS/JS
- 標準機能: グラフビュー、バックリンク、全文検索、ポップオーバープレビュー、SPA ルーティング、Latex、シンタックスハイライト、RSS、i18n
- 構成: TypeScript + Preact + esbuild。設定は `quartz.config.ts` と `quartz.layout.ts`
- 要件: Node.js v22+、npm v10.9.2+
- 思想: 「個人のノートをネットワーク状の知識サイトとして公開する」ことに特化

---

## Q2. GitHub Pages 運用のシンプルな Markdown プロジェクトに適用できる？

**A.** できる。ただし「Markdown 直配信」から「ビルドステップ入り」へ運用が変わる。

**移行手順:**
1. `jackyzha0/quartz` を fork または clone
2. `content/` に既存の `.md` を移動
3. `quartz.config.ts` の `baseUrl` を自分のドメインに変更
4. `.github/workflows/deploy.yml` を追加(公式テンプレあり)
5. Settings → Pages の Source を "GitHub Actions" に変更

**注意点:**
- URL 形式が変わる(`foo.html` 形式 → 既存リンクが壊れる可能性)
- README 数枚だけならオーバースペック

## Q3. 今のプロジェクトは「Obsidian 記法は使わないが、記事間はリンクでつないでいる」。Quartz は合う?

**A.** 半分以上の旨味を享受できる。

**動くこと:**
- 通常の Markdown リンク `[text](./other.md)` は `CrawlLinks` プラグインで処理される
- バックリンク、グラフビューも Markdown リンクから自動生成される
- ポップオーバープレビューも機能する

**注意点:**
- 画像の絶対パスは baseUrl の関係で調整が要るかも
- 既存 URL の互換性は崩れる
- frontmatter なしでも動くが、タイトルを綺麗に出したいなら付け足したほうがよい

**結論:** 思想としては Quartz 寄り。ただし機能てんこ盛りでやや過剰感はある。

---

## Q4. Quartz と MkDocs の違いは(Obsidian 以外で)?

**A.** 設計思想が根本的に違う。

| 観点 | MkDocs | Quartz |
|---|---|---|
| **構造** | ツリー型(`nav:` で章立て) | ネットワーク型(リンクで偶然出会う) |
| **UX** | 地図(サイドバー)を見て歩く | リンクをたどって発見する |
| **速度** | Instant loading で速い | SPA ルーティングでより滑らか |
| **ランタイム** | Python | Node.js v22+ |
| **Markdown 処理** | Python-Markdown | unified/remark/rehype(現代最強級) |
| **設定** | `mkdocs.yml` 一枚で完結 | TypeScript ファイル + JSX 風レイアウト |
| **カスタマイズ** | 設定で済む範囲が広い | ソース全部手元、深いカスタマイズも可 |
| **見た目** | 技術文書として洗練 | 個人サイトとして洗練、よりモダン |

**Quartz 標準で MkDocs にないもの:** バックリンク、グラフビュー、ポップオーバープレビュー、OG 画像自動生成

**MkDocs Material 強み:** 構造化 nav、バージョン切替、成熟した i18n

**使い分け:**
- 順序立てて読ませる技術文書 → MkDocs Material
- リンクで横断する読み物・個人サイト → Quartz

## Q5. Quartz 寄りで、もう少し軽いジェネレーターは?

**A.** 主な候補は 4 つ。

### 候補 1: Eleventy + デジタルガーデンテンプレ ⭐おすすめ
- Node.js だが Quartz よりずっと薄い
- バックリンク対応のテンプレが複数あり(`eleventy-garden` など)
- 設定は `.eleventy.js` 一枚
- グラフビューやポップオーバーは標準ではなし

### 候補 2: Astro
- 島アーキテクチャでデフォルト JS ゼロ
- 見た目のモダンさで Quartz とタメを張れる唯一の選択肢
- 自由度が高い反面、組み立てる部分が出てくる

### 候補 3: Hugo
- Go バイナリ一個で動く、Node も Python も不要
- ビルド速度は Quartz の 10 倍以上
- テンプレートが Go template で独特の学習コスト

### 候補 4: Zola
- Rust バイナリ一個、設定は TOML 一枚
- 依存ゼロ、起動も生成も極速
- バックリンクはテーマ依存

### 比較表

| | ランタイム | 依存量 | 設定の重さ | バックリンク | グラフ | モダン度 |
|---|---|---|---|---|---|---|
| **Quartz** | Node v22+ | 重い | TS 2ファイル | ◎ 標準 | ◎ 標準 | ◎ |
| **Eleventy** | Node | 軽い | JS 1ファイル | テンプレ次第 | △ | ○ |
| **Astro** | Node | 中 | TS/JS | テンプレ次第 | △ | ◎ |
| **Hugo** | Go バイナリ | 極小 | TOML/YAML | テーマ次第 | △ | テーマ次第 |
| **Zola** | Rust バイナリ | 極小 | TOML | テーマ次第 | × | テーマ次第 |

## Q6. あなたの状況でのおすすめは?

**A.** 優先順は以下。

1. **Eleventy + デジタルガーデンテンプレ** — Quartz から過剰な部分を引いたちょうど良い場所。Node 系で揃えたいならここ
2. **Astro** — 見た目のモダンさで Quartz と並ぶ唯一の選択肢。自分で組み立てる気があるなら最強の自由度
3. **Hugo** — 「Node もう要らない、とにかく軽く」ならこれ。見た目モダンを求めるならテーマ選びに一手間

「Quartz の主要機能を残しつつ、ビルド時間と依存の重さを半分以下に」したいなら **Eleventy のデジタルガーデンテンプレ** が一番ハマる可能性が高い。
