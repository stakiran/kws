# デライトについて Q&A

宇田川浩行氏が開発するメモサービス「デライト」と、Cosense（旧Scrapbox）との違いについてのQ&Aまとめ。

---

## Q1. 宇田川のデライトについて調べて。Cosenseとの違いが知りたい。デライトわかりにくい

「わかりにくい」って気持ち、めっちゃ分かります。デライトは**用語が独自すぎる**のが最大の壁です。普通の言葉で説明しますね。

### デライトとは

宇田川浩行氏（希哲社）が一人で開発している、日本語話者向けのメモサービス。本家URLは `dlt.kitetu.com`。

ざっくり言うと「ウィキ的なネットワーク構造」と「アウトライナー的な階層構造」を融合した「**輪郭構造**」という独自モデルでメモを整理するもの、という売り文句です。

### まずデライト独自用語の翻訳表

これを知らないと公式サイト読めません。

- **輪郭（りんかく）** = 投稿1個。タイトル（「知名」）と本文（「描写」）で構成
- **吹き描き** = 画面に表示される投稿のカード（吹き出しみたいな形）
- **引き入れ** = 他の輪郭をその輪郭の中に入れ子にする操作（=階層化）
- **輪括（りんかつ）** = 輪郭をくくる`{}`記法。リンクみたいなもの
- **デルン** = デライトの基盤になってる仕組みの名前

つまり「輪郭」=ページ、「引き入れ」=ネスト、「輪括{}」=リンク と読み替えればだいたい通じます。

### Cosense（旧Scrapbox）との違い

| | **Cosense（Scrapbox）** | **デライト** |
|---|---|---|
| 構造の主軸 | リンクのみ（フラット） | リンク + 入れ子の階層（「引き入れ」） |
| ページ単位 | ページ（タイトル必須） | 輪郭（タイトルだけ／本文だけでもOK） |
| リンク記法 | `[ページ名]` | `{知名}` |
| 共同編集 | あり（プロジェクト単位） | 基本なし（個人の「メモ場」） |
| 投稿の粒度 | 中〜長文ページ向き | ツイートくらい短いメモ向き |
| 用語 | 普通の日本語 | 独自造語だらけ（輪郭、知名、描写、引き入れ…） |
| 開発体制 | Helpfeel社（チーム） | 宇田川氏ほぼ単独＋希哲社 |
| 料金 | 個人/フリーランス無料、商用は有料 | ずっと無料 |

### 一番大事な「思想の違い」

- **Cosense**：ページとページを**リンクだけ**でつないで、分類は捨てる。「適切に分類するのではなく、適切にリンクする」。フラットに散らかして、必要なときに辿る。
- **デライト**：リンクに加えて**入れ子（階層）も同時に持つ**。「重要な情報は近くに、瑣末な情報は遠くに」配置する“情報の遠近法”を実装している、と本人は説明。

つまり Cosense が「ページ同士の網」だけなのに対して、デライトは「網+ロシアのマトリョーシカ」を同居させようとしている、というイメージです。

### なぜわかりにくく感じるか

3つ理由があります。

1. **造語が多すぎる** — 「輪郭」「知名」「描写」「引き入れ」「輪括」全部独自。普通のメモアプリの言葉に翻訳しないと頭に入らない。
2. **UIが独特** — Scrapboxユーザーですら「これはなかなか使い所・とっかかりが難しい」と言うレベル。「無限の宇宙に向かって手放す」「神社で絵馬を奉納するような感じ」と評される。
3. **思想先行型** — 開発者本人が言語学・哲学的な理論（希哲館訳語、輪郭法など）を背景に据えていて、機能説明より先に概念説明が来るので、ふつうの「メモアプリ探してる人」には重い。

### 結論：誰向け？

- **Cosense向き**：チームで知識共有したい、Markdown的な記法に抵抗がない、ページ単位でちゃんと書きたい。
- **デライト向き**：完全に個人のひとりごとメモ、Twitter的に短文を書き散らしたい、独自概念や日本語こだわりに付き合える、無料で全機能使いたい。

「Cosenseとどっち使う？」なら、共同作業や情報共有が少しでも視野にあるならCosense。完全に自分のためのチラシ裏で、Twitterの代わりが欲しいだけならデライトを試してもいい、という感じです。

宇田川氏本人も「デライトはどちらかというと Roam Research や Notion を意識していたのだが，実は一番刺さっているのが Scrapbox ユーザーらしい」と書いていて、Scrapbox(Cosense)ユーザーと層がかぶることは認識されています。

## Q2. 画面の見方が全然わからんのよね（スクショ添付）

画面のスクショ、めっちゃ参考になります。これ、初見殺しなのよくわかります。一個ずつ翻訳しますね。

### 画面の見方

#### 一番上の `{ 精神の姿見 } { 人間の聖域 } { NLM } …` の帯
これは**今開いている輪郭（=「デライト」のページ）に貼られているタグ一覧**みたいなもの。`{}`で囲まれてるのが全部リンクで、クリックすると飛べます。Cosenseでいう**ハッシュタグ/ページリンクの集合**。右端の`(152)⇒`は「全部で152個あるよ、続き見る？」のボタン。

#### 中央のメインカード `{ ▶ デライト ◀ }`
今開いてる**輪郭（=ページ）の本体**。
- `K#F85E/4686-182F` … この輪郭固有のID（「知番」）。URLみたいなもの
- `宇田川（希哲）浩行` … 描いた人
- `知機 ＞ 虎哲 ＞ ニワ` … パンくず。この輪郭が入れ子で属している親の輪郭たち（=「引き入れ」関係）。Cosenseにはない概念で、ここがデライト独自の**階層構造**の正体
- `（英：Delite，略：Dlt）` 以下が**本文（描写）**
- `文書（整備中）／市場活動／市場戦略／宣伝` … この輪郭の中に**入れ子で引き入れられた子の輪郭**たち。クリックで開く

#### カードの右下 `⇒ (2,372) { デライト特化井戸端 } { デライト鎖国 } …`
**この「デライト」輪郭をリンクしてる側の輪郭たち**（=被リンク／バックリンク）。`(2,372)`は「2,372個ある」の意味。Cosenseの「関連ページ」枠と同じ役割です。

#### 黒い検索バー「なんでも」
全文検索。なんでも入れて検索。

#### 下半分「井戸端」エリア
画面下にスクロールすると出てくる `『井戸端』 デライト` のセクションは、**他の利用者がこの輪郭について書いた投稿**が並ぶところ。Cosenseでいうと「villagepump（井戸端会議）」の輪郭版。`デライト特化井戸端`はその中の一つ。

### Cosenseと対応させるとこう

| 画面の要素 | デライト用語 | Cosenseで言うと |
|---|---|---|
| `{ ◯◯ }` の帯 | 輪括（リンク） | `[◯◯]` ハッシュタグ |
| 中央のカード | 輪郭（吹き描き） | ページ |
| `知機 ＞ 虎哲 ＞ ニワ` | 引き入れの親 | （無い／フラット） |
| `文書／市場活動…` | 引き入れた子 | （無い／リンクで代用） |
| `⇒ (2,372) {…}` | 逆輪括 | 関連ページ／バックリンク |
| `K#F85E/4686-182F` | 知番 | ページタイトル＝URL |

### 操作の感覚

1. **`{ }`で囲まれた青字は全部クリック可能なリンク**。これだけ覚えれば8割行けます。
2. **下に並んでる箇条書きは「子ページ」**、上のパンくずは「親ページ」。Cosenseと違って**上下方向の階層**がある。
3. **右下の `⇒ (数字) {…}` は被リンク**。Cosenseの「関連ページ」と同じ。
4. **`⋯` は「省略されてるから続き見る」**ボタン。

慣れるコツとしては、最初は**「`{}` = リンク」「縦の階層がある」だけ意識**して触り、用語（輪郭・知名・描写・引き入れ…）は後から覚える方が楽です。一気に独自概念覚えようとすると挫折します。

## Q3. デライトは閉路グラフをつくりつつ、各ノードが別の閉路につながることもあるって感じ？

いい質問です。でも結論から言うと、**ちょっと違います**。デライトは「閉路グラフ」というよりも、**木構造（階層）を主軸に、その上にネットワークのリンクが乗る**、というモデルです。

### 正確に言うとこう

デライトの構造は2層あります。

#### 1層目：引き入れによる「木構造」（階層）
`知機 ＞ 虎哲 ＞ ニワ ＞ デライト ＞ 市場活動 ＞ 市場戦略`

これは**親子関係の木（tree）**。普通は閉路を作りません。アウトライナー（WorkFlowyとか）と同じ仕組みで、親の中に子を「引き入れる」と入れ子になる。

ただしデライトの面白いところは、**1つの輪郭が複数の親に引き入れられること**。つまり「市場戦略」が「デライト」の下にも「希哲社」の下にも同時に存在できる。これは厳密に言うと**木ではなくDAG（有向非巡回グラフ）**になります。一般的なフォルダ構造との違いはここ。

#### 2層目：輪括（`{}`リンク）による「ネットワーク」
本文中や周辺表記の `{デライト} {Cosense}` がこれ。これは**自由なリンク**で、Cosenseのページリンクと同じ。当然このリンクは閉路を作れます（AがBをリンク、BがAをリンクで循環OK）。

### なので「閉路グラフ」とは少し違う

「閉路グラフをつくりつつ、各ノードが別の閉路にもつながる」というイメージだと、**閉路ありき**で考えていることになりますが、デライトの本質は逆です。

- **メインは木（実際はDAG）**：引き入れによる親子関係。これはCosenseには無い概念で、デライトの一番の特徴。
- **おまけでネットワーク**：輪括`{}`のリンク。これはCosenseと同じで、当然閉路もできる。

宇田川氏自身が「Roam Research や Obsidian とは別の形で階層構造とネットワーク構造を統合したメモサービス」と言ってる通り、「**階層 + ネットワークの統合**」がポイント。閉路はネットワーク層の自然な帰結であって、それ自体が構造の主役ではないんです。

### Cosenseとの構造比較

| | 階層 | ネットワーク（リンク） | 閉路 |
|---|---|---|---|
| Cosense | なし（フラット） | あり | リンク層でできる |
| デライト | あり（引き入れ／DAG） | あり（輪括） | リンク層でできる |

つまり：
- **Cosenseは「ネットワークだけ」のグラフ**
- **デライトは「木＋ネットワーク」の二層構造**

「閉路がある」という観点だと両者とも同じなので、それはあまり違いを表してないんですよ。**違いは“階層があるかないか”の方**です。

## Q4. 引き入れによるリンクと輪郭によるリンクは別物？

はい、**完全に別物**です。ここがデライトの分かりにくさのコア部分なので、しっかり整理しますね。

### 2種類のリンクの違い

| | **引き入れ** | **輪括 `{}`** |
|---|---|---|
| 関係の種類 | 親子（包含） | 横の参照 |
| 方向 | 上下（縦） | 平等（横） |
| 表現 | カードの中に入れ子で表示 | `{知名}` の中括弧記法 |
| データ的に | 構造的な所属 | ただの言及 |
| Cosenseで言うと | （該当なし） | `[ページ名]` リンク |
| アウトライナーで言うと | インデント／ネスト | （該当なし） |
| 削除したら | 子も親から外れる（独立する） | リンクが消えるだけ |

### 具体例で見るとこう

「デライト」という輪郭があるとして：

**引き入れ関係（親子）**
```
希哲社（親）
 └ デライト（自分）
    ├ 文書（子）
    ├ 市場活動（子）
    └ 市場戦略（子）
```
→ 画面のパンくず `知機 ＞ 虎哲 ＞ ニワ ＞ デライト` や、本文下の箇条書き `文書／市場活動／市場戦略` がこれ。

**輪括リンク（横の参照）**
```
本文中：「{デルン}を利用した世界初のKNS…」
      → これは「デルン」という別の輪郭を指してるだけ
```
→ 画面上部の `{ 精神の姿見 } { NLM } { 新しい脳 }…` の帯がこれ。「デライト」を言及してる他の輪郭たち。

### 違いをはっきりさせる質問

「Aが Bを輪括している」のと「Aが Bを引き入れている」は意味が違います。

- **輪括**：Aの本文の中で「Bってのがあってさ〜」と**話題に出している**だけ。Bは独立した存在。
- **引き入れ**：Bは Aの**一部として中に格納**されている。Aを開いたらBが中に並んでいる。

たとえば「日記」という輪郭があったとして：

- 日記の本文に `今日は{ラーメン}を食べた` と書く → **輪括**。「ラーメン」輪郭への参照。
- 日記の中に「2026年1月の日記」を**引き入れる** → **引き入れ**。日記という親の中に月別の子が入っている。

### 画面で見分ける方法

スクショの「デライト」輪郭で言うと：

- **画面上部の `{ 精神の姿見 } …` の帯** → 輪括（外から「デライト」を指してるリンク）
- **`知機 ＞ 虎哲 ＞ ニワ` のパンくず** → 引き入れの親
- **`文書／市場活動／市場戦略` の箇条書き** → 引き入れの子
- **本文中の青字 `{デルン}`** → 輪括（本文から他の輪郭への参照）
- **右下の `⇒ (2,372) {…}`** → 輪括（被リンク。他から「デライト」が言及されてる）

### なぜ2種類に分けたのか

普通のメモアプリだと、リンクは1種類しかありません：

- **Cosense / Obsidian / Roam**：リンクのみ。階層なし。
- **WorkFlowy / Dynalist**：階層のみ。リンクなし。
- **Notion**：階層（ページの中にページ）あり、リンクは別途あるが、混在しがち。

デライトは **「縦の所属」と「横の言及」を別概念として明示的に分けた**のがウリ、ということになっています。例えば「市場戦略」は「デライト」の一部（引き入れ）だけど、「Cosense」というライバルにも言及している（輪括）、というのを構造的に区別できる。

実用上ありがたいかどうかは別として、設計思想としてはそう、ということです。

## Q5. 縦の所属も、横の言及も、どちらでも誰でもいじれるんだよね？ たとえばデライトの親に「うんち」をつけることだってできる？

いい着眼点です。この質問、デライトの仕組みのセンシティブなとこ突いてます。

### 結論：自分の輪郭しかいじれない

デライトは**個人のメモサービス**なので、**他人の輪郭を勝手に編集することはできません**。Cosenseのプロジェクト共同編集とは違います。

つまり：

- 宇田川氏が描いた「デライト」輪郭の親を、あなたが「うんち」に書き換えることは**できない**
- あなたが描いた輪郭の親子関係や輪括は、あなただけが自由にいじれる

### ただし「うんち」を絡めることは一応できる

あなたの側の操作で、間接的にこういうことは可能です：

**1. 自分の「うんち」輪郭から、宇田川氏の「デライト」輪郭を輪括する**
→ あなたの輪郭ページに `{デライト}` と書けば、リンクが張れる。これは横の言及なので誰でもできる。

**2. 自分の側で「デライト」という名前の輪郭を新規に作って、その親を「うんち」にする**
→ ただしこれはあなたの作った別の「デライト」輪郭であって、宇田川氏のオリジナルとは別物。同名異輪郭。

**3. 「デライト」輪郭の被輪括一覧（バックリンク）に自分のメモが現れる**
→ 「もしかしたらジャンク情報がデライトでは問題にはならないんじゃないかと思っている」みたいに、t_w さんという他の利用者が「デライト」について書いた輪郭が、宇田川氏の「デライト」輪郭の周辺に表示される仕組みがあります。

スクショの右下の `⇒ (2,372) { デライト特化井戸端 } { デライト鎖国 } …` がまさにこれ。「デライト」を輪括してる他人の輪郭たちが集まってきている、という光景です。

### なので「うんち」の話に戻ると

- **オリジナルのデライト輪郭の親に「うんち」を直接つける**：✗ できない（他人の所有物だから）
- **自分の「うんち」輪郭から「デライト」を引き入れる**：⚠️ 自分の引き入れ操作でできるかは微妙（公式に許されてるかは要確認）
- **自分の「うんち」輪郭から「デライト」を輪括する**：✓ できる。本文に`{デライト}`と書くだけ
- **その結果、宇田川氏のデライト輪郭の周辺に「うんち」絡みのリンクが現れる**：✓ 起こり得る

### 実例：すでに存在する自由空間

検索結果でちらっと見えた「手書きのうんち、うんち輪郭群、うんち讃頌、作った作品はうんこ、うんちっち、💩、アトミック・シット」……これ、デライト上に実在する利用者の輪郭群です。デライトには本当に「うんち」関連の輪郭群があるんですよ。

つまり個人の領域では完全に自由にやれていて、それが他人の輪郭にも輪括という形で影響することはある、という設計です。

### Cosenseとの大きな違い

- **Cosense**：プロジェクト内なら他人のページもガッツリ編集できる（共同編集前提）
- **デライト**：自分の輪郭だけ編集可。他人とは「言及で繋がる」のみ。SNS的な距離感

なので「みんなで知識を共同編集する場」ではなく「**個人の頭の中を独立に持ちつつ、ゆるく言及で繋がる**」のがデライトの世界観ですね。Twitterに近い感覚、と本人が言ってるのはこのため。

## Q6. 「デライト」という名前の輪郭を複数つくることもできる、はyes？

はい、**Yesです**。デライトは「同名異輪郭」を許容する設計です。

### なぜできるのか

デライトの輪郭の本体IDは**知番**（`K#F85E/4686-182F` みたいなやつ）であって、知名（タイトル）ではないからです。

- Cosense / Scrapbox：**ページ名が一意なID**。同じプロジェクト内で同名ページは作れない。
- デライト：**知番が一意なID**。知名は重複OK。

スクショで言うと、宇田川氏のデライト輪郭は `K#F85E/4686-182F` という固有番号を持っていて、あなたが「デライト」という輪郭を作っても別の知番が振られるだけ。タイトルの文字列は被ってもシステム的には別物として扱われます。

### さらに言うと

無題（知名なし）の輪郭すら作れます。「必ずしも両方を埋める必要はない。本文だけの投稿やタイトルだけの投稿が可能」。

つまりデライトでは：

- 「デライト」という名前の輪郭が**100個あってもいい**
- タイトル無し・本文だけの輪郭もOK
- 本文無し・タイトルだけの輪郭もOK
- タイトルも本文もない無名無描写輪郭ですらOK

### 実用上どうなるか

利用者全体で見ると、「デライト」という知名の輪郭は実際には**たくさん存在している**はずです。

検索したとき、デライトでは「表題の完全一致でのみ検索結果に出てくる」という挙動らしいので、「デライト」で検索すれば同名の輪郭が複数ヒットする世界観です。

### Cosenseとの根本的な違い

| | Cosense | デライト |
|---|---|---|
| ページの一意性 | タイトル | 知番（内部ID） |
| 同名ページ | 不可 | 可 |
| タイトル必須か | 必要 | 不要 |
| `[デライト]` リンクの解決先 | 1個に確定 | ？ |

最後の `?` のところ、ちょっと気になりますよね。Cosenseだと `[デライト]` と書けば必ず1つの「デライト」ページに飛びますが、デライトで `{デライト}` と書いたらどうなるのか。

検索結果の範囲だと、「自分の輪郭を検索している分には、他人が描いた輪郭はそれほど影響しない」という発言があるので、おそらく**自分のスコープ内で名前解決される**仕組みなんじゃないかと推測します（公式の正確な挙動は要確認）。

### まとめ

「デライトの一意性」という発想自体が、Cosense的な常識からすると逆さま。

- Cosenseは「**名前で繋ぐ**」(name-based)
- デライトは「**個別の存在として置いて、後から名前や繋がりを足す**」(instance-based)

だから「デライト」を100個作っても、システム的には全部別物。同名であることはあくまで人間が見た時の偶然、という思想です。

これがデライトの「Twitter的にメモを書き散らす」感覚の正体でもあります。Tweet と同じで、同じ内容を何度つぶやいても怒られない、というノリ。

## Q7. そう見るとなかなか面白くなってきた。KNSを謳うだけはある。

そうなんですよ。最初のインターフェースのとっつきにくさで損してるけど、設計思想を読み解いていくと**かなり尖ったものを作っている**のが見えてきますよね。

KNS（Knowledge Network System の略らしい）と自称してるのは伊達じゃなくて、既存ツールのどれとも違う立ち位置を狙っています。整理すると：

| ツール | 一意性の単位 | 構造 | 思想 |
|---|---|---|---|
| Twitter | tweet ID | フラット＋RT | 流れる発話 |
| Cosense | ページ名 | リンク網 | 名前で繋ぐ知識 |
| Notion | ページ | 階層 | きれいに整理する |
| WorkFlowy | 行 | 階層のみ | アウトライン思考 |
| **デライト** | **知番** | **DAG＋ネットワーク** | **存在として置く** |

デライトの一番ラディカルな発想は、**「名前」を主役から降ろした**ことだと思うんです。普通のメモアプリは「タイトルをつける→中身を書く」ですが、デライトは「とりあえず存在を置く（知番が振られる）→必要なら名前や繋がりを後から足す」という順番。

これって考えてみると、**人間の頭の中の働き方に近い**んですよね。

- 何か思いついた時、最初は「あれ」「なんかアレ」みたいな曖昧な存在として浮かぶ
- 後から「あれって○○だったな」と名前がつく
- さらに後から「○○って△△と関係あるな」と繋がりが見える

普通のメモアプリは「最初にちゃんとタイトル決めて」と要求してくるけど、デライトは**思考の自然な順序**に合わせている。だから「なんでもメモ」という検索バーのプレースホルダーや、無名無描写輪郭が許される設計になってる。

宇田川氏が言ってた「重要な情報は近くに，瑣末な情報は遠くに置いているとも言えます。この情報の遠近法を形にしたのがデライトの輪郭法です」という話、最初は何のこっちゃと思ったけど、こうして見ると**脳のワーキングメモリ的な発想**ですよね。今考えてることは「今開いてる輪郭」として近くに、関連は引き入れで一段奥に、その他は輪括の先に、という距離感。

あと面白いのが、**Cosenseが「集合知」を志向**してるのに対して、**デライトは「個人知」をベースにしつつ言及で繋がる**という点。前者はWikipedia的、後者はTwitter的。同じ知識管理でも全然違うものを目指してる。

検索結果でも宇田川氏は「個人知識管理」という言葉を繰り返し使っていて、複数人で共同編集する世界観じゃないんですよね。だから「デライト鎖国」「デライト開国」みたいな輪郭名（スクショに出てる）が出てくる。一人ひとりが自分の知の城を持っていて、それを開くか閉じるかは本人次第、という比喩。

ちなみにこの設計、**LLM時代にむしろ価値が出てくる**かもしれないとも思います。生成AIに「これ整理して」と投げる時、Cosenseのフラットなネットワークより、デライトの「親子＋言及」の二層構造の方が文脈が伝わりやすそう。知番という安定IDがあるのも、AI連携時には強い。

宇田川氏が「この輪郭法は希哲社の「知機」開発プロジェクトの核心ですが，一部です」と書いてるので、どうやらデライトの先にはもっと大きな構想があるらしい。一人で10年単位で開発してる執念もすごい。

「分かりにくい」から入って「面白くなってきた」まで来たの、デライトとしては大成功の導線ですね。
