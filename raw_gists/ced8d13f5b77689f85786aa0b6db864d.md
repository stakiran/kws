# tipsboard：作者が力を入れている部分の分析

作者（kyu999）が tipsboard で最も魅力的・注力している部分はどこか。コミット履歴・CHANGELOG・コード規模から推定した。

## 結論

力を入れている所は2つあり、層が違う。

| | 何 | 性格 |
|---|---|---|
| **① 最も“工数”を注いだ核** | **インライン装飾エディタ（書き心地そのもの）** | 地味だが土台。一番の労力 |
| **② 最も“差別化”を狙った目玉** | **ローカル・セマンティック検索** | 売り・自慢ポイント |

加えて、**最近イテレーションが激しいのが Canvas**。ただし本人も「experimental」と明言しており、まだ実験段階。

## ① 本命：書き心地を支える「インライン装飾エディタ」

`tipsboard-decorations.ts` が **全ファイル中ダントツ最大の1676行**。CodeMirror の `WidgetType` / `Decoration` を使い、**生の Markdown を編集可能なまま、その場で見栄えだけ整える**仕組み。

- `MarkdownImageWidget` / `TableWidget` / `MermaidWidget` / `KaTeXWidget`（画像・表・図・数式をその場レンダリング）
- `BulletWidget`、bold / italic / link / tag / quote / code のマーク装飾
- `[ ]` リンク判定（`tipsboard-links.ts`）

カーソル安定化に何リリースも費やし（`cursor-test-fixtures.ts` 558行＋専用テスト多数）、Cosense/Obsidian と真っ向勝負する「ストレスなく書ける感触」を作り込んでいる。**一番の地道な工数がここ**。

## ② 目玉：ローカル・セマンティック検索

README で破格に大きく扱われ、`semantic.ts`(862) + `semanticRuntime.ts`(292) + 専用コマンド3つ + `eval/semantic/` 評価環境まで持つ。

- Transformers.js をローカル実行、`Xenova/multilingual-e5-base` で意味検索
- hybrid（dense + BM25）ランキング + リランク
- **オフライン対応**（閉域網向けにランタイム/モデルを手動配置）
- 「Nearby Notes（意味的に近いノート）」として関連発見にも再利用

vscode-scb にも素の Obsidian にもない**明確な差別化ポイント**。「ローカル・サインイン不要なのに意味検索」が自慢ネタ。

## 補足：Canvas は“今まさに賭けている”が未確定

0.3.9 → 0.4.0 → 0.4.1 は Canvas 一色。しかも 0.4.0 で実装ごと丸replace（無限ボード → Mermaid 問題構造エディタ）。熱量は高いが「experimental」で旧フォーマット非互換を切るほど作り直し中。**「推し」というより実験段階の野心**と読むのが正確。
