# 日本語Markdownを英訳する

`$ARGUMENTS` で指定されたMarkdownファイルを英訳し、同じディレクトリに `<basename>_EN.md` として保存する。

## 使い方

- `/en path/to/foo.md` のようにファイルパスを指定する
- `$ARGUMENTS` が空のときは、対象ファイルパスをユーザーに尋ねる

## 手順

1. `$ARGUMENTS` から対象ファイルパスを決定する
   - 絶対パス・相対パスどちらも受け付ける
   - 空のときは IDE で開いているファイルのパスを使う
2. `Read` ツールで対象ファイルの内容を読み取る
3. 下記「翻訳方針」に従って自然な英語に翻訳する
4. 出力ファイルパスを決定する
    - 入力 `path/to/foo.md` → 出力 `path/to/foo_EN.md`
    - 入力ファイルと同じディレクトリに配置する
5. `Write` ツールで出力ファイルに**翻訳後のMarkdown本文のみ**を保存する
    - 原文は含めない
    - 全体をコードブロックで囲わない（生の Markdown として書き出す）
6. 保存したファイルの絶対パスをユーザーに報告する

## 翻訳方針

あなたはプロのバイリンガルライターとして、日本語 Markdown を自然な英語に翻訳する。以下を厳守する。

- 元の Markdown 構造（見出し、リスト、コードブロック、引用など）を保持する
- サブリストのインデントは必ず 4 space で出力する
    - たとえばこのようになる（深さ2）
        - たとえばこのようになる（深さ3）
            - たとえばこのようになる（深さ4）
- 本文中で定義されている訳語を優先的に使用する
- 引用記法中の空行はそのまま保持する
    - `> ` には意味がある。スペースも含んでいる。このまま使う。`>` のように Right Trim しない
- 見出し行の次の行には空行を入れない
- `YYYY/MM/DD` 形式の日付文字列は ISO 8601 形式（`YYYY-MM-DD`）に変換する。英語圏で読みやすくするため
- 出力ファイルの内容は翻訳後の Markdown のみとする。翻訳元の原文を含めない
- 出力全体を ```markdown ``` などのコードブロックで囲わない
