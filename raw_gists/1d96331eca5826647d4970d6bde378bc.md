# q
強い男性エンジニアは太っている。

この仮説、どう思います？
根拠をいうと、

- 長時間デスクで集中しているので運動量が足りない、しかし頭は使うので摂取は多い
- 不養生な生活でも長時間の作業を可能とするだけの生命力（身体の強さ、病気との相性など総合的な強さ）がある
    - また生命力があるので欲求も強く、通常パートナーか、ペットか、その両方がいることも多い

# a
面白い仮説ですね。観察としては「あるある」感がありますが、論理構造を見ると、いくつか突っ込みどころがあります。

まず一番気になるのは、2つの根拠が逆方向を向いている点です。1つ目は「環境が太らせる」(座りっぱなし+摂取過多→太る)という因果の話で、2つ目は「太っていても耐えられる生命力がある」という選択効果の話。前者だと「強いエンジニアだから太る」、後者だと「太っても潰れない人だけが強いエンジニアとして残る」になり、実は別の仮説です。後者は生存者バイアスの構造ですね。不養生な生活で身体を壊した人は「強いエンジニア」として観測される前に脱落するので、残った集団に頑健な人が多く見えるのは自然です。

次に、交絡が大きそうです。年齢(シニアで強い人は年齢層が高く、加齢で代謝が落ちる)、収入(強いエンジニアは高収入→外食・デリバリーが増える)、労働時間など、「強さ」と「体型」の両方に効く変数がいくつもあるので、直接の関係があるように見えても媒介変数経由の可能性が高い。

あと「頭を使うので摂取が多い」は弱い根拠です。脳の消費カロリーは集中してもほとんど増えないことが知られていて、デスクワーカーの過食はどちらかというとストレス・退屈・習慣によるものです。なので「頭を使うから食べる」より「座って手が空くから食べる」の方が説明として正確かもしれません。

「生命力があるので欲求も強く、パートナーやペットがいる」のくだりは、正直かなり飛躍しています。性欲・食欲・愛着行動・免疫的な頑健さを「生命力」という一つの潜在変数にまとめてしまっているので、反証不可能な物語になりやすい。これは仮説というより人物類型(キャラクター造形)に近いです。

反例もすぐ思いつきます。痩せ型で異常に強いエンジニア、筋トレ趣味の強いエンジニアはどの職場にも一定数いますよね。むしろ体型は二極化していて、「強さ」と相関するのは体型そのものより「自分のリソース配分に無頓着でいられる度合い」なのでは、という対抗仮説も立てられます。太るのも、逆にストイックに鍛えるのも、コードに全振りした結果の異なる表現型、という見方です。

まとめると、「強いエンジニアの集団に太った人が目立つ」という観察自体は否定しませんが、それは強さの証拠というより、生活環境・生存者バイアス・記憶に残りやすさ(恰幅のいい強い人は印象的)の合成だと思います。仮説として面白いのは確かなので、検証するなら「強さ」の定義(コミット数?年収?周囲の評価?)とBMIのデータを取って、年齢と労働時間で調整してみたいところですね。

# q
そうなんですよ。

> 「強いエンジニアの集団には太った人が目立つ」

この傾向が明らかにあると思っていて、なぜそうなのかを突き止めたいんです。

# a
いいですね、それなら「観察が正しいか」と「正しいとしたら何のメカニズムか」を分けて詰めていきましょう。

まず観察そのものの検証から。「明らかにある」と感じる時こそ怪しい、というのが認知バイアスの常で、候補は3つあります。1つ目は利用可能性ヒューリスティック。「強くて太っている人」は2つの目立つ属性が重なるので記憶に残りやすく、「強くて普通体型の人」は体型が記憶にタグ付けされない。後から思い出すと前者ばかり浮かびます。2つ目はベースレート無視。日本の中年男性の肥満率はもともと3割前後あるので、エンジニア集団で「太った人が目立つ」のは単に母集団の反映かもしれない。3つ目は確証バイアスで、仮説を持った瞬間から該当例ばかりカウントし始める。これを潰すには、自分の知っている「強いエンジニア」を先に強さ基準だけでリストアップして、その後に体型を思い出して集計する、という順序が大事です。体型から思い出すと汚染されます。

その上で、傾向が本当にあると仮定した場合のメカニズム候補を整理すると、だいたい5系統に分かれると思います。

環境因果説。長時間の座位、不規則な食事、睡眠不足(睡眠不足は食欲ホルモンを直撃します)、ストレス。これは「強い」エンジニアほど裁量と没頭時間が長いので曝露量が大きい、という量の問題として説明できます。あなたの1つ目の根拠の精緻化版ですね。

リソース配分説。認知資源と意志力を全部コードに割いていて、体型管理に配分する気がない。これは「強さ」の本体である没頭力・優先順位の極端さが、副作用として体型に出るという話です。因果が「強さ→太る」ではなく「同じ性質→強さと太るの両方」になっている点が環境因果説と違います。

シグナリング/文化説。ハッカー文化には伝統的に「外見に構わないことが本物の証」という価値観があって、太っていることや無頓着な身なりが、逆説的に実力への信頼を高める。つまり太った人が「強く見える・強く評価される」方向の因果。これだと実際の実力分布は体型と無関係でも、あなたの観測は成立します。

生存者バイアス説。前回話した、不養生に耐えた頑健な個体だけが残る説。あなたの2つ目の根拠です。

逆因果説。先に太っていた(あるいは運動が苦手だった)人が、屋内・座位・一人で完結する活動に幼少期から流れ、結果的にコンピュータへの蓄積時間が長くなって強くなった。1万時間の獲得経路として体型が先、という説です。これは個人的にかなり有力だと思っていて、「中学生の頃から外で遊ぶよりPCの前にいた」という強いエンジニアの典型的な来歴と整合します。

面白いのは、これらは予測が違うので原理的には区別できることです。例えば逆因果説が正しければ、太ったのは強くなる前のはずなので、本人たちの体型史を聞けばわかる。環境因果説なら、エンジニアになってから太ったはず。シグナリング説なら、客観的な実力指標(コードの質など)と体型は無相関で、評判ベースの「強さ」とだけ相関するはず。生存者バイアス説なら、若手では相関が弱く、ベテランほど強く出るはず。

なので次の一手としては、あなたの周りの「強いエンジニア」たちについて、いつから太っているか・若手とベテランで傾向が違うかをまず観察してみるのが安上がりで筋がいいと思います。どの説が本命だと感じていますか?
