# 会話をMarkdownとして保存

このスキルが呼び出されたら、ユーザーの指示に従って会話の内容をMarkdownファイルに保存する。

## 使い方

ユーザーは `$ARGUMENTS` で保存する範囲や対象を指定する。例:
- `/mm ここまでの議論`
- `/mm ここまでの議論をここまで対話してきたとおりに`
- `/mm サーバーサイド化に関するやりとりだけ`

## 手順

1. `$ARGUMENTS` の指示に従い、保存対象の範囲・内容を特定する
2. ファイル名を生成する
   - 形式: `YYYYMMDD_HHMMSS_short-topic-slug.md`
   - トピックスラッグは対象内容の主題から英語で短く生成する
   - 日時は `date +%Y%m%d_%H%M%S` で取得する
3. 保存先ディレクトリを準備する
   - `Bash` で `mkdir -p ~/memo` を実行する
4. `Write` ツールで `~/memo/` にMarkdownファイルを保存する
   - **必ず `Write` ツールを使うこと**（Bashのechoやcatでは日本語・絵文字が文字化けする可能性がある）
   - ファイルパスは `$HOME/memo/` 以下の絶対パスで指定する（`$HOME` は `Bash` で `echo $HOME` して取得する）
5. 保存したファイルを `Bash` で `start "" "<パス>"` を実行し、関連づけられたアプリで開く
6. 保存したファイルパスをユーザーに報告する

## ガイドライン

- 日本語で書くこと
- 会話の要約ではなく、指示された範囲の内容を忠実に再構成すること
- Markdownの見出し・リスト・コードブロックなどを適切に使い、読みやすく整形すること
