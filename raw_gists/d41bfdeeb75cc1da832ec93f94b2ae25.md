# ASUS ROG Strix PC 動作遅延トラブルシューティング記録

## 環境

- **機種**: ASUS ROG Strix(ノートPC)
- **CPU**: Intel Core i7-10750H @ 2.60GHz
- **GPU**: NVIDIA GeForce RTX 2060 (6GB GDDR6)
- **RAM**: 15.8GB
- **OS**: Windows 11

## 症状

朝、休止状態からの復帰後に発生:

- 画面描画(反映)が重い
- プログラムの起動が遅い
- Python CLIツールの起動にも時間がかかる
- Minecraftの起動が明らかにおかしい
- Windows Update適用・再起動でも改善せず
- タスクマネージャ上、プロセスに目立った異常なし

## 切り分けの過程

### 仮説1: NVIDIAドライバの不具合(誤り)

2日前にNVIDIAドライバを更新していたため、当初これが原因と推測。

- 現在のドライバ: 596.49(2026/05/12リリースの最新版相当)
- ドライバのロールバックを試みるも、デバイスマネージャの「ドライバーを元に戻す」ボタンはグレーアウト
- クリーンインストール用にStudio Driver 596.36(2026/04/28)をダウンロード

→ **後の調査により、ドライバは無関係と判明**

### 仮説2: 電源モードの問題(部分的)

タスクマネージャのパフォーマンスタブで以下を確認:

- **CPUクロック: 0.80GHz**(i7-10750Hのベース2.60GHzに対し異常に低い)
- CPU温度: 41℃(冷えている)
- ファン: 表示上2500RPMだが実質ほぼ無音
- GPU: アイドル状態

Windows電源モードが「バランス」になっていたため「最適なパフォーマンス」に変更。Armoury Crateも「パフォーマンス」モードに設定。

→ **CPUクロックは798MHzのまま変化なし**。ソフトウェア側の設定では制御できない、より低レイヤーの問題と判断。

### 仮説3(真犯人): ASUSシステムファームウェアの不具合

MyASUSのシステム診断を実行:

- ハードウェア異常: 0件
- 解決案: 3件提示
- そのうち1件が「**ASUSTeK COMPUTER INC. - Firmware - 10.1.2.314**」のWindows Update経由配信

このファームウェアはEmbedded Controller(EC)を含むシステム制御を司る部分のため、以下を制御する:

- 電源管理(CPUクロック制御を含む)
- バッテリー充放電制御
- ファン制御
- 熱センサー周り

旧バージョンのファームウェアに、**休止状態からの復帰時にCPU電力制限が誤った状態で固定される**不具合があったと推測される。

## 解決手順

1. **Windows Update → オプションの更新プログラム** を開く
2. **ASUSTeK COMPUTER INC. - Firmware - 10.1.2.314** をインストール
3. 再起動 → BIOSアップデート画面が自動起動
4. 「Yes/No」ダイアログは60秒で自動的にアップデート開始
5. Warning画面表示後、アップデート進行(電源は絶対に切らない)
6. 自動再起動を経てWindowsログイン画面へ復帰

所要時間: 約5〜15分

## 結果

| 項目 | 修正前 | 修正後 |
|---|---|---|
| CPU周波数 | 798 MHz | **4284 MHz** |
| CPU温度 | 41℃(冷えすぎ) | 61℃(健全) |
| CPU使用率 | 12% | 32% |
| GPU周波数 | 300 MHz | 1005 MHz |
| CPUファン回転数 | 2500 RPM(空回り) | 4700 RPM |
| GPUファン回転数 | 2500 RPM | 4100 RPM |
| ファン音響 | 24.9 dBA | 37.8 dBA |

- 画面描画の重さ → 解消
- プログラム起動の遅延 → 解消
- Python CLI起動の遅延 → 解消
- Minecraftの起動 → 正常に戻った

## 今回の学び

### 切り分けの教訓

「直近で変更した作業(NVIDIAドライバ更新)」は直感的に犯人候補に挙げやすいが、**同時に別の問題(BIOS/ファームウェア更新の必要性)が進行している可能性もある**。タイミングが重なっただけで、必ずしも原因とは限らない。

### 効果的だった切り分け手法

1. **タスクマネージャのパフォーマンスタブでCPUクロックを確認** — 一番強力な切り分け指標
2. **温度・ファン回転数・使用率を総合的に見る** — 一つの数字だけでなく整合性をチェック
3. **MyASUSのシステム診断を実行** — ハードウェア起因か否かの切り分けに有効
4. **Windows Updateのオプション更新を確認** — メインの更新と別にドライバ・ファームウェア更新が隠れていることがある

### CPU動作の正常範囲(参考)

- アイドル時に800MHz前後 → 省電力動作で正常
- 負荷がかかった瞬間に2GHz以上にブースト → 正常
- 負荷をかけても800MHz固定 → 異常(電力制限・熱制限などを疑う)

### 同様の症状が再発したら

ASUS ROGノートで、休止/スリープ復帰後に急に動作が重くなった場合、以下の順で確認:

1. タスクマネージャでCPUクロックを確認(800MHz固定なら電源管理系を疑う)
2. ファンの実際の動作音を物理的に確認
3. ASUS/Windows Updateで未適用のファームウェア・BIOS更新がないか確認
4. MyASUSのシステム診断を実行

## 結論

**真の原因**: ASUSシステムファームウェアの旧バージョンにおける、休止状態復帰後のCPU電力管理不具合

**解決策**: Windows Update経由のオプション更新「ASUSTeK COMPUTER INC. - Firmware - 10.1.2.314」適用
