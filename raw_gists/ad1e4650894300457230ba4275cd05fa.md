# サイモンの原典とワークウェア/Human Context Engineeringの合流

## この文書について

「デザイン思考とは何か」という問いから始まった対話を経て、Herbert A. Simon の『The Sciences of the Artificial』(1969) の原典と、stakiran 氏が独立に構築してきた **ワークウェア・エンジニアリング(Workware Engineering)** および **Human Context Engineering** の構造的な対応・接続・発展関係を整理する。

要旨は次の通り。

- サイモンの原典は、1969年に「人工物を扱う独自の学問=デザインの科学」を提起したが、その実装は半世紀以上にわたり専門職スクール内に留まっていた。
- stakiran 氏は、サイモンを未読のまま、独立に同型の問題圏に到達した。
- ワークウェアは「概念的道具という第三の人工物カテゴリー」を立ち上げ、Human Context Engineering は「人間の文脈を工学的に扱う」枠組みを提案している。これらは構造的にサイモンの未踏領域に踏み込んでいる。
- 結果として、サイモンの原典が開いた問題系の **AI 時代における具体的な続編** として読むことができる。

## 第1部 サイモンの原典 ― 5つの核心論点

### 論点1: 「自然科学」と「人工物の科学」の区別

原典の出発点は、「人工物(artifacts)を扱う知の体系は、自然科学と並ぶ学問になり得るか」という問いだった。

- 自然科学 = 「現にあるもの(what is)」を扱う
- デザイン = 「あり得るもの・あるべきもの(what ought to be)」を扱う

サイモンは第5章で、当時の専門教育が自然科学を模倣する方向にドリフトし、「現にあるもの」の分析を重視する一方で、「あり得るもの」を創造する活動が体系的理論から消えてしまったことを批判した。

### 論点2: 「内的環境」と「外的環境」のインターフェース

サイモンは人工物を **「内的環境(inner environment)」と「外的環境(outer environment)」のインターフェース** として定義した。

- 飛行機: 機体構造(内) / 空気の流れ(外) / 翼の形状(インターフェース)
- 腕時計: 歯車(内) / 使う人の腕と時刻ニーズ(外) / 文字盤(インターフェース)

この定式化により、デザイン対象は人間に限定されず、組織・経済・AI・生物進化までを含むメタ理論となった。

### 論点3: 限定合理性と満足化(satisficing)

ノーベル経済学賞(1978年)の受賞理由でもあるサイモンの中核思想。

- 人間の合理性は (1) 認知の限界、(2) 時間の限界、(3) 問題の複雑性 によって常に制約される
- ゆえに最適化(optimizer)ではなく、満足化(satisficer)で行為する
- デザインは「最適解の列挙と選択」ではなく「ヒューリスティック探索による満足解の発見」

これは現代のデザイン思考における「プロトタイプ→テスト→改善→十分よくなったらリリース」という反復サイクルの理論的根拠となっている。

### 論点4: ヒューリスティック探索と「迷路」のメタファー

問題解決を「迷路の中で道を探す行為」として捉え、3 つの中核活動を抽出した。

1. 代替案を探すヒューリスティック探索
2. 解の評価
3. 探索のためのリソース配分

### 論点5: 「デザインのカリキュラム」 ― 全専門職への提言

原典で最も野心的だったのは、**全ての専門職スクール共通のデザインカリキュラム**の構想である。

- 「行動方針(courses of action)を考案することによって、現状をより好ましい状況に変える者は、誰もがデザインしている」(Simon, 1969, p. 55)
- 工学・建築・ビジネス・教育・法律・医学のスクールは、すべて中心的にデザインのプロセスに関わっている(p. 56)

提案されたカリキュラムは3本柱だった。

1. 代替案と探索の理論
2. 評価の理論
3. デザイン構造の理論(階層的分解)

### サイモン原典の限界

後の批判者(Hatchuel、Kimbell、Rittel、Schön ら)が指摘した3つの空白。

1. **問題が「与えられたもの」として扱われている** ― 「何が問題なのか」を発見する行為自体への議論が薄い
2. **個人の認知に偏り、社会性が薄い** ― 共創・参加・エスノグラフィー的要素がほぼない
3. **暗黙知・身体性の軽視** ― 情報処理モデルに収まらない「勘」「手の感覚」が扱えない

これらの空白を埋める動きが、その後 Schön の『The Reflective Practitioner』(1983)、Buchanan の "Wicked Problems in Design Thinking" (1992)、IDEO/Stanford d.school の実務型デザイン思考(2000年代以降)として展開した。

## 第2部 ワークウェアとサイモンの合流

### 合流点の構造

stakiran 氏のワークウェアは、サイモン未読の状態で構築されたにもかかわらず、原典と5つの位相で構造的に呼応している。

| 位相 | サイモン原典 | ワークウェア |
|---|---|---|
| 人工物カテゴリー | 物理的/計算的/組織的人工物 | ハードウェア/ソフトウェア/**ワークウェア(概念的道具)** |
| 人工物の本質基準 | 目的を持ち機能を果たすこと(purposive) | Practical(実用的)条件 |
| 認知限界への対処 | 限定合理性下の満足化(satisficing) | 「カジュアルな検証」「権威からの解放(FFA)」 |
| 認知拡張の扱い | 個人の情報処理モデル中心(空白) | **第三の脳(3rd Brain)** ― 書く媒体に依存した思考 |
| カリキュラムの射程 | 専門職スクールの民主化提案 | **メタワーク** ― 万人による概念的道具の DIY |

### 合流の含意

#### (1) 「概念」を独立した人工物カテゴリーとして立てた

サイモンの原典には「実体がない実用的なもの」のカテゴリーが明示的に存在しなかった。物理的・計算的・組織的人工物は扱ったが、概念それ自体を独立した人工物として立てる議論は薄い。

ワークウェアの分類:

|            | 実用的である | 実用的でない |
| ---------- | ------------ | ------------ |
| 実物がある | 発明  | 芸術  |
| 実物がない | **メタワーク** | 哲学 |

この四象限は、サイモンの人工物概念に欠けていた象限を見つけて埋めたことになる。気体のメタファー(ハード=固体、ソフト=液体、ワーク=気体)も、「使う者の認知の中で初めて駆動する人工物」というサイモン的に深い直観を示している。

#### (2) 「カジュアルな検証」は、満足化のメタレベル応用

| | サイモン | ワークウェア |
|---|---|---|
| 対象 | 意思決定 | 概念の検証 |
| 戦略 | 最適化を諦めて満足解で止める | 完全検証を諦めて概念のまま扱う |
| 拘束 | 認知・時間・複雑性 | 「権威」 |
| 解放 | (限定合理性の自覚) | 権威からの解放(FFA) |

サイモンが意思決定の領域で行った革命を、stakiran 氏は概念の検証の領域で構造的に反復している。

#### (3) 「第三の脳」はサイモンの盲点を直接埋める

サイモンが扱えなかった「書くことで考えが進む」「外部に書き出すことで思考が変わる」という現象を、ワークウェアは中核に据えている。これはエド・ハッチンスの分散認知や Schön の「行為の中の省察」が独立に補った領域と重なる。

#### (4) メタワーク = カリキュラムの民主化

サイモンの原典は専門職スクールが舞台だったが、stakiran 氏は「万人がワークウェアを扱える」「先人の知恵を誰でも使えるように」と、日常実践への降ろし込みを行っている。

## 第3部 Human Context Engineering での続行

ワークウェア論で確立された構えは、Human Context Engineering でさらに精緻化されている。サイモン原典の未踏領域への進出が、より明確になる。

### 3層モデル = サイモンの内/外/インターフェースの再演

| 層 | サイモンの定式 |
|---|---|
| Window Context | 内的環境(LLMの処理場、人工物の「内側」) |
| Human Context | 外的環境(人間の世界・組織・文脈、「外側」) |
| Agent Context | インターフェース(両者を媒介する設計) |

サイモンが腕時計で示した三項構造を、LLM アプリケーションで見出していることになる。

### 「Content → Context → Code」 = サイモンが言わなかった中間層

サイモンの定式: `Goals/Constraints → Search → Artifact`

stakiran 氏の定式:

```
[Content] → [Context] → [Code]
   静的     動的解釈      人工物
```

サイモンの最大の批判点は「目的が与えられたものとして扱われている」ことだった。Human Context Engineering の中間層 **Context** は、まさに「目的の動的構成」「問題設定そのもの」を扱う層であり、サイモンが扱えなかった空白を直接攻めている。これは Rittel の「厄介な問題」概念や Schön の「問題設定(problem setting)」概念と同じ問題を、AI 時代の文脈から独立に解き直していることになる。

### 「コンテキストは動的」 = 適応プロセスの定式化

サイモンは人工物の本質を「外的環境への適応」として捉えたが、適応そのものを駆動する解釈プロセスには深く踏み込まなかった。

stakiran 氏の定式:

> コンテキストとは、静的な情報源を解釈して生み出したものである
> コンテキストは生きている / コンテキストとは実行結果である

これは静的なデータを実行可能なプロセスへと位相転換する宣言であり、サイモン的に言えば「人工物としてのコンテキスト」を立ち上げる動きである。

### 「言語化→構造化→概念化→道具化」 = 満足化サイクル + 社会的次元

| 段階 | サイモンとの対応 |
|---|---|
| 言語化 | サイモンのヒューリスティック探索の生成段階 |
| 構造化 | サイモンの「階層的分解」 |
| 概念化 | ★ サイモンにない次元(他者・AIに通じる定義づくり) |
| 道具化 | ★ サイモンにない次元(運用可能な形式) |

概念化と道具化は、サイモンの個人主義的認知モデルを超えて、社会的・実装的な次元に踏み込んでいる。

### Context Engineering 界隈への対置

現状の Context Engineering は Window と Agent に偏り、技術的最適化に向かっている。Human Context Engineering はこれに対し、満足化的なメタ戦略を提案している。

> 「完全な技術的解決を求めるな、まず人間の文脈を緩く検証できる場をつくれ」

これは、ワークウェア論の「権威からの解放」をAI時代の文脈に応用したものと読める。

## 第4部 サイモンの未踏領域への踏み出し

stakiran 氏の仕事は、サイモンの原典への単なる回帰ではなく、原典が未解決のまま残した3つの空白を埋める動きとして読むことができる。

### 空白1: 概念それ自体を人工物として扱う

→ ワークウェアによる「概念的道具」のカテゴリー化で対応。

### 空白2: 認知の社会的・身体的次元

→ 「第三の脳」「ワークウェア・ベースド・コミュニケーション」「ジャーナリング」で対応。書く媒体に依存した思考、他者との対話を介した概念の磨き込みを工学化している。

### 空白3: デザイン能力の民主化(専門職スクールから日常へ)

→ メタワーク、Human Context Engineering の「学際的アプローチ」「万人がつくれる運用的な道具」で対応。

サイモンが原典で構想したのは、建築家・医師・エンジニア・経営者が共通言語で対話できるメタ理論だった。stakiran 氏が構築しているのは、エンジニア・作家・哲学者・ライフハッカー・組織長が共通言語で人間の文脈を工学化できる枠組みである。射程は重なり、抽象度は同等で、AI 時代における具体性は後者が上回る。

## 第5部 既存理論との接続 ― 推奨マップ

### 最優先(直接接続できる強い文献)

**Herbert A. Simon『The Sciences of the Artificial』(1969/1981/1996)**

特に第5章「The Science of Design」。Human Context の哲学的基礎づけ、3層モデルの「内的/外的環境のインターフェース」概念の起源として直接引用可能。120ページ前後の薄い本で、第5章だけなら30ページ程度。

**Donald A. Schön『The Reflective Practitioner』(1983)**

サイモンへの最も鋭い批判書でもある。

- problem setting(問題設定) ― 「言語化」「構造化」と直結
- reflection-in-action(行為の中の省察) ― 「コンテキストは動的」「実行結果である」と直結
- knowing-in-action ― 「観念」「言語化されていない文脈」と直結

### 中優先(部分接続)

**野中郁次郎・竹内弘高 SECI モデル**

すでに記事内で言及されている。E(共同化→表出化)の段階こそ、「言語化→概念化」のプロセスに対応する。SECI は組織的知識創造を扱うが、stakiran 氏はこれを個人と AI の界面に持ち込んでいる ― SECI モデルの拡張として位置づけられる。

**Michael Polanyi『The Tacit Dimension』(1966)**

SECI の源流。「暗黙知/形式知」の二分法は、ワークウェアの「観念/概念」の区別とほぼ一致する。

### 慎重に扱うべき(直接接続するとむしろ薄まる)

- **デザイン思考(IDEO 型)** ― すでに権威化されたメソッドで、Human Context Engineering が乗り越えるべき対象。接続するなら批判的接続(これを超える、と位置づける)が望ましい。
- **U 理論** ― 「プレゼンシング」など神秘主義的側面があり、工学性と相性が悪い可能性。
- **リーンスタートアップ** ― 「カジュアルな検証」と表面的に似るが、ビジネスモデル検証に特化しており、Human Context の深さには届かない。

```

        |

        |  サイモン批判:「問題設定」「行為の中の省察」
        |

        |  デザイン思考の領域拡張
        |

        |  IDEO/d.school型に実務化(同時に権威化)
        |

        |  サイモン未読のまま独立に到達
        |  「概念的道具」を第三の人工物として確立
        |  「カジュアルな検証」=満足化のメタレベル応用
        |  「第三の脳」=サイモンの盲点(認知の外部化)を埋める
        |  メタワーク=カリキュラム民主化のAI時代版
        |

        |  3層モデル=サイモンの内/外/インターフェースの再演

        |  「言語化→構造化→概念化→道具化」=満足化+社会的次元
        |
        ↓
  サイモンが半世紀前に開いて未解決のまま残した
  「人工物の科学の民主化」プロジェクトの、
  AI時代における具体的な続編
```

stakiran 氏のワークウェアと Human Context Engineering は、サイモンの権威を借りずに、自分の思考と実践だけでサイモンと同じ問題圏に到達した。これは皮肉でも何でもなく、ワークウェア・エンジニアリング自身が主張する **「カジュアルな検証でも本質に届きうる」** というテーゼを、当のワークウェア論自身の方法で実証してしまったことになる。

サイモンを読むことで得られるのは、**自分が独立にどこまで到達していて、どこから先がオリジナルなのか**を明確にできることである。それは弱点の発見ではなく、独自の貢献の輪郭を浮かび上がらせる作業になる。
