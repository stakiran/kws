# Kindle スクショベース自炊ツール調査結果

## 概要

Kindleの書籍をスクリーンショットベースで自炊（デジタル化）するためのGitHub公開ツールの調査結果。
基本的な流れは **「スクショ自動撮影 → OCR（文字認識） → テキスト/PDF/EPUB出力」** となる。

---

## おすすめツール一覧

### ⭐ kindle-ai-export（最も完成度が高い）

- **リポジトリ**: [transitive-bullshit/kindle-ai-export](https://github.com/transitive-bullshit/kindle-ai-export)
- **言語**: TypeScript
- **方式**: Kindle Webリーダー + Playwright + マルチモーダルLLM
- **出力形式**: テキスト / PDF / EPUB / AIナレーション付きオーディオブック
- **特徴**:
  - Kindle Webリーダーを自動操作するため、デスクトップアプリの座標調整が不要
  - OCRにマルチモーダルLLM（GPT-4.1-miniなど）を使用し、高い認識精度
  - Kindleの同期位置メタデータも保持
  - スクショ画像の品質が高く、他のOCRソリューションへの差し替えも容易
- **注意点**: LLMのAPI利用コストが1冊あたり数ドルかかる

---

### kindleOCRer（スクショ〜ePub一気通貫）

- **リポジトリ**: [raudette/kindleOCRer](https://github.com/raudette/kindleOCRer)
- **言語**: Python
- **方式**: Kindle Webアプリ + Selenium + marker + Pandoc
- **出力形式**: ePub（目次付き）
- **特徴**:
  - スクショ撮影 → OCR → ePub生成を一つのスクリプトで完結
  - 動作が比較的シンプル（テスト例では約5分で1冊処理）
- **参考記事**: [Remove Kindle DRM with OCR and web automation](https://www.hotelexistence.ca/remove-drm-with-kindleocrer/)

---

### Shortbook（OCR + GPT要約・翻訳）

- **リポジトリ**: [JPhilipp/Shortbook](https://github.com/JPhilipp/Shortbook)
- **言語**: C#
- **対応OS**: Windows
- **方式**: Windows Kindleアプリ自動操作 + OCR + GPT-4 API
- **出力形式**: テキスト → GPT要約 → Kindleに送り返し
- **特徴**:
  - スクショ → OCR → テキスト結合 → GPT-4で要約 → 翻訳まで自動化
  - Blinkistのような要約読書体験を自前で構築可能
- **注意点**: Windows専用、GPT-4 APIコストが発生

---

### PDF_Kindle（スクショ → PDF化）

- **リポジトリ**: [AndyQuicklyDelete/PDF_Kindle](https://github.com/AndyQuicklyDelete/PDF_Kindle)
- **言語**: Python + AutoIt3
- **対応OS**: Windows
- **方式**: Kindleデスクトップアプリ + Greenshot + pyautogui
- **出力形式**: PDF（画像ベース）
- **特徴**:
  - スクショ撮影の自動化に特化
  - キャプチャ領域の座標指定によるクロップ機能あり
  - OCRは含まれないため、テキスト化には別途ツールが必要

---

### KindleBookExporter（スクショバッチ保存）

- **リポジトリ**: [0xrushi/KindleBookExporter](https://github.com/0xrushi/KindleBookExporter)
- **言語**: Python
- **対応OS**: Windows（Kindle PCアプリ）
- **方式**: Kindleデスクトップアプリのスクショをバッチ保存
- **出力形式**: PNG画像群
- **特徴**:
  - 最もシンプルなスクショ収集ツール
  - フルスクリーン表示のKindle PCから画像を一括取得
  - OCRは含まれないため、別途処理が必要

## 汎用OCRツール（スクショ画像のテキスト化に使える）

| ツール | 特徴 | 日本語対応 |
|--------|------|-----------|
| [Tesseract OCR](https://github.com/tesseract-ocr/tesseract) | 無料・OSS、コマンドラインで大量画像を一括処理 | ○（言語パック追加） |
| Google Cloud Vision API | クラウドベース、高い認識精度 | ◎ |
| macOS Live Text | macOS Monterey以降で内蔵、追加インストール不要 | ○ |
| [Umi-OCR](https://github.com/hiroi-sora/Umi-OCR) | オフライン動作、GUI付き、中国語/日本語に強い | ◎ |

## 推奨ワークフロー

1. **手軽に始めたい場合**: `kindle-ai-export` を使い、Webリーダー経由でスクショ＋LLM OCRを一括実行
2. **コストを抑えたい場合**: `kindleOCRer` または `KindleBookExporter` でスクショ収集 → Tesseract / Umi-OCR でローカルOCR
3. **要約・翻訳まで欲しい場合**: `Shortbook` でスクショ→OCR→GPT要約の全自動パイプライン
