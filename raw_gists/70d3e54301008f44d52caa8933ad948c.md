# エグゼクティブサマリー：AI時代の開発プロセス改革

## 一言で言うと

AIで「つくる」コストが激減した今、開発のボトルネックは実装力ではなく「何をつくるか」の見極めに移った。探索フェーズを個人主体で設計し直すことで、この変化に対応する。

## 現状の課題

- 実装速度がAIで上がった分、方向が間違った開発を高速に回すリスクが増大
- 合議ベースの探索は無難な方向に収束しやすく、本質的な発見につながりにくい
- 「手を動かす＝生産的」という前提が、上流の思考への投資を阻害している

## 提案の骨子

| | 探索ループ（前半） | 実行フェーズ（後半） |
|---|---|---|
| 目的 | 何をつくるべきかを見極める | 確信を持ったものを磨き上げる |
| 体制 | 個人（1人1テーマ＋AI） | チーム（従来型） |
| 評価 | 集団レビューで選別 | 通常のQA・レビュー |
| 反復 | 確信が得られるまで複数回 | スプリント等で段階的に |
| マネジメント | キュレーション型 | 進捗管理型 |

## 期待される効果

- 探索の多様性と並列度が向上し、より筋の良いテーマを発見できる
- 個人の主体性と納得感が高まり、エンジニアの成長とエンゲージメントにつながる
- 実行フェーズ開始時点での方向性の確度が上がり、手戻りコストが減少する

## リスクと対策

- 探索が発散しすぎる → 集団レビューの定期開催で収束の場を確保
- 個人の探索力にばらつきがある → マネージャーによる問いの立て方の支援
- 組織文化との摩擦 → 小規模な試行から開始し、段階的に拡大
