# Windows に Node.js をインストールする

このガイドでは、Windows 開発環境に Node.js をインストールする方法を説明します。

Linux 環境で Node.js を使用したい方は、「[Windows Subsystem for Linux (WSL2) に Node.js をインストールする](https://learn.microsoft.com/ja-jp/windows/dev-environment/javascript/nodejs-on-wsl)」をご覧ください。

## ネイティブ Windows と Linux (WSL 2) のどちらで開発するか

Node.js をインストールして開発する際に、ネイティブ Windows 環境と Linux (WSL 2) 環境のどちらを使うか検討するにあたって、以下の点を考慮してください。

- **スキルレベル**: Node.js での開発が初めてで、すぐに動かして学びたい場合は、Windows に Node.js をインストールしてください。Windows での Node.js のインストールと使用は、WSL を使うよりも初心者にとってシンプルです。
- **コマンドラインツール**: PowerShell を使いたい場合は、Windows 上の Node.js を使用してください。Bash を使いたい場合は、Linux (WSL 2) 上の Node.js を使用してください。
- **本番サーバー**: Node.js アプリを Windows Server にデプロイする予定であれば、Windows 上の Node.js を使用してください。Linux サーバーにデプロイする予定であれば、Linux (WSL 2) 上の Node.js を使用してください。WSL では好みの Linux ディストリビューション（デフォルトは Ubuntu）をインストールでき、開発環境（コードを書く場所）と本番環境（コードがデプロイされるサーバー）の一貫性を確保できます。
- **パフォーマンスとシステムコールの互換性**: Linux と Windows のパフォーマンスについては議論や開発が続いていますが、Windows マシンを使用する際に重要なのは、Node.js をインストールしたファイルシステムと同じファイルシステムにプロジェクトファイルを保存することです。Windows ファイルシステムに Node.js をインストールした場合は、ファイルを Windows ドライブ（例: C:/）に保存してください。Linux ディストリビューション（Ubuntu など）に Node.js をインストールした場合は、使用しているディストリビューションに関連付けられた Linux ファイルシステムのディレクトリにプロジェクトファイルを保存してください。（WSL ディストリビューションのコマンドラインから `explorer.exe .` と入力すると、Windows エクスプローラーでディレクトリを参照できます。）
- **Docker コンテナ**: Windows 上で Docker コンテナを使ってプロジェクトを開発したい場合は、[Docker Desktop on Windows をインストール](https://docs.docker.com/desktop/setup/install/windows-install/)することをお勧めします。Linux ワークスペースで Docker を使用する場合は、Linux と Windows の両方のビルドスクリプトを保守する必要がないように、「[WSL 2 で Docker Desktop for Windows をセットアップする](https://learn.microsoft.com/ja-jp/windows/wsl/tutorials/wsl-containers)」を参照してください。

## nvm-windows、Node.js、npm のインストール

Windows と WSL のどちらにインストールするかに加えて、Node.js のインストール時に追加の選択肢があります。バージョンは非常に速く変わるため、バージョンマネージャーの使用をお勧めします。作業中のプロジェクトのニーズに応じて、複数の Node.js バージョンを切り替える必要が出てくるでしょう。Node Version Manager（通称 nvm）は、複数バージョンの Node.js をインストールする最も一般的な方法ですが、Mac/Linux 専用で Windows ではサポートされていません。代わりに nvm-windows をインストールし、それを使って Node.js と Node Package Manager (npm) をインストールすることをお勧めします。次のセクションで説明する[代替バージョンマネージャー](#代替バージョンマネージャー)も検討対象です。

> **重要**: バージョンマネージャーをインストールする前に、OS にインストール済みの Node.js や npm を削除することを強くお勧めします。異なるインストール方法が混在すると、奇妙で紛らわしい競合が発生する可能性があります。既存の Node.js インストールディレクトリ（例:「C:\Program Files\nodejs」）の削除も含まれます。NVM が生成するシンボリックリンクは、既存の（空であっても）インストールディレクトリを上書きしません。以前のインストールの削除方法については、「[How to completely remove node.js from Windows](https://stackoverflow.com/questions/20711240/how-to-completely-remove-node-js-from-windows)」を参照してください。

> **警告**: NVM はユーザーごとにインストールし、シェルごとに呼び出すように設計されています。共有の開発マシンや、複数のビルドエージェントを持つビルドサーバー向けには設計されていません。NVM はシンボリックリンクを使用して動作します。共有環境で nvm を使用すると、そのリンクがあるユーザーの AppData フォルダーを指すため問題が発生します。ユーザー X が `nvm use lts` を実行すると、マシン全体の node リンクがそのユーザーの AppData フォルダーを指すようになります。ユーザー Y が node や npm を実行すると、ユーザー X のアカウント下のファイルが実行され、`npm -g` の場合はユーザー X のファイルが変更されますが、デフォルトではこれは許可されていません。そのため、nvm は個人の開発マシンでのみ使用してください。ビルドサーバーも同様で、2つのビルドエージェントが同じ VM/マシン上にある場合、競合が発生しビルドで予期しない動作が起こる可能性があります。

### インストール手順

1. [nvm-windows リポジトリ](https://github.com/coreybutler/nvm-windows#installation--upgrades)のインストール手順に従ってください。インストーラーの使用をお勧めしますが、より高度な理解がある場合は手動インストールも検討できます。インストーラーは最新バージョンの[リリースページ](https://github.com/coreybutler/nvm-windows/releases)を案内します。
2. 最新リリースの **nvm-setup.zip** ファイルをダウンロードします。
3. ダウンロードしたら、zip ファイルを開き、**nvm-setup.exe** ファイルを開きます。
4. Setup-NVM-for-Windows インストールウィザードが、nvm-windows と Node.js のインストール先ディレクトリの選択を含む、セットアップ手順を案内します。
5. インストールが完了したら、PowerShell を開き（管理者権限での起動を推奨）、nvm-windows を使って現在インストールされている Node のバージョンを確認します（この時点では何もないはずです）: `nvm ls`
6. Node.js の最新リリースをインストールします（最新機能のテスト用ですが、LTS よりも問題が発生する可能性が高いです）: `nvm install latest`
7. Node.js の最新安定 LTS リリースをインストールします（推奨）。まず `nvm list available` で現在の LTS バージョン番号を確認し、次に `nvm install <version>`（`<version>` をバージョン番号に置き換え、例: `nvm install 12.14.0`）で LTS バージョンをインストールします。
8. インストール済みの Node バージョンを一覧表示します: `nvm ls` — インストールした 2 つのバージョンが表示されるはずです。
9. 必要な Node.js バージョンをインストールしたら、使用するバージョンを選択します: `nvm use <version>`（`<version>` をバージョン番号に置き換え、例: `nvm use 12.9.0`）
10. プロジェクトで使用する Node.js のバージョンを変更するには、新しいプロジェクトディレクトリを作成し（`mkdir NodeTest`）、そのディレクトリに入り（`cd NodeTest`）、`nvm use <version>` と入力します（`<version>` を使用したいバージョン番号に置き換え、例: v10.16.3）。
11. インストールされている npm のバージョンを確認します: `npm --version`。このバージョン番号は、現在の Node.js バージョンに対応する npm バージョンに自動的に変わります。

### 代替バージョンマネージャー

NVM for Windows (nvm-windows) は現在最も人気のある Node バージョンマネージャーですが、以下の代替手段も検討できます。

- **[nvs](https://github.com/jasongin/nvs)**（Node Version Switcher）: クロスプラットフォームの nvm 代替で、[VS Code との統合](https://github.com/jasongin/nvs/blob/master/doc/VSCODE.md)機能があります。
- **[Volta](https://github.com/volta-cli/volta#installing-volta)**: LinkedIn チームによる新しいバージョンマネージャーで、速度の向上とクロスプラットフォームサポートを特徴としています。

Volta をインストールするには、[Getting Started ガイド](https://docs.volta.sh/guide/getting-started)の **Windows Installation** セクションに進み、Windows インストーラーをダウンロードして実行し、セットアップ手順に従ってください。

Windows で Volta を使用して複数バージョンの Node.js をインストールする方法については、[Volta Docs](https://docs.volta.sh/guide/understanding#managing-your-toolchain) を参照してください。

## Visual Studio Code のインストール

Windows で Node.js を使って開発する場合は、[Visual Studio Code のインストール](https://code.visualstudio.com)をお勧めします。詳しくは「[Node.js tutorial in Visual Studio Code](https://code.visualstudio.com/docs/nodejs/nodejs-tutorial)」を参照してください。

### 代替コードエディタ

Visual Studio Code 以外のコードエディタや IDE を使いたい場合、以下も Node.js 開発環境として適しています。

- [WebStorm](https://www.jetbrains.com/webstorm/download/#section=windows)
- [Sublime Text](https://www.sublimetext.com/3)
- [Atom](https://atom.io/)
- [Brackets](http://brackets.io/)
- [Notepad++](https://notepad-plus-plus.org/)

## Git のインストール

他の人と共同作業したり、プロジェクトをオープンソースサイト（GitHub など）でホストする予定がある場合、VS Code は [Git によるバージョン管理](https://code.visualstudio.com/docs/editor/versioncontrol#_git-support)をサポートしています。VS Code の「ソース管理」タブは、すべての変更を追跡し、一般的な Git コマンド（add、commit、push、pull）を UI に組み込んでいます。まず Git をインストールして、ソース管理パネルを有効にする必要があります。

1. [git-scm ウェブサイト](https://git-scm.com/download/win)から Git for Windows をダウンロードしてインストールします。
2. インストールウィザードが Git インストールの設定に関する一連の質問をします。特別な理由がない限り、すべてデフォルト設定の使用をお勧めします。
3. Git を使ったことがない場合は、[GitHub Guides](https://guides.github.com/) が始め方の参考になります。
4. Node プロジェクトに [.gitignore ファイル](https://help.github.com/en/articles/ignoring-files)を追加することをお勧めします。こちらが [GitHub の Node.js 用デフォルト gitignore テンプレート](https://github.com/github/gitignore/blob/master/Node.gitignore)です。

## Windows Server での Node.js

Windows サーバーで Node.js アプリをホストする必要がある（やや稀な）状況では、最も一般的なシナリオはリバースプロキシの使用のようです。これには 2 つの方法があります: 1) iisnode を使用する方法、2) 直接リバースプロキシを設定する方法です。Microsoft はこれらのリソースをメンテナンスしておらず、Node.js アプリのホスティングには [Linux サーバーの使用](https://learn.microsoft.com/ja-jp/azure/app-service/app-service-web-get-started-nodejs)を推奨しています。
