# 特定プロジェクトでのみプラグインを有効にする方法（Claude Code）

## 背景

- Marketing Skills リポジトリ（`coreyhaines31/marketingskills`）をプラグインとして入れたい
- ただしユーザーグローバルに入れると、全セッションで description 分の context を食うので避けたい
- 「使いたいプロジェクトに限定」したい

## プラグインのインストールは2段階

**Stage 1: マーケットプレイスの登録**

```
/plugin marketplace add coreyhaines31/marketingskills
```

→ `.claude-plugin/marketplace.json` を読みに行って「このリポジトリにこういうプラグインがあるよ」というカタログを登録するだけ。まだ何もインストールされない。

**Stage 2: プラグイン本体のインストール**

```
/plugin install marketing-skills
```

→ 登録済みマーケットプレイスから指定プラグインを取得してユーザー領域に展開。ここで初めてスキルが使える状態になる。

### ポイント

- マーケットプレイス = プラグインの**索引**（GitHubリポジトリ等）
- プラグイン = 実体（skills / commands / hooks / agents / MCP の詰め合わせ）
- 1つのマーケットプレイスに複数プラグインが含まれうる
- 管理コマンド: `/plugin marketplace list`、`/plugin list`、`/plugin uninstall <name>`、`/plugin marketplace remove <name>`

## 結論: プロジェクト単位の制御は「半分」できる

**本体の「置き場所」は必ずグローバル**（`~/.claude/plugins/`）。プロジェクトローカルに閉じたインストールはできない。

ただし**「どのプロジェクトで有効化するか」はプロジェクト側で制御可能**。

| 項目 | 可否 |
|------|------|
| インストール先をプロジェクトに限定 | 不可（必ず `~/.claude/plugins/`） |
| プロジェクト単位で有効/無効を切替 | 可（`enabledPlugins`） |
| マーケットプレイスをプロジェクト単位で登録 | 可（`extraKnownMarketplaces`） |

## プロジェクト単位で有効化する書き方

プロジェクトの `.claude/settings.json`:

```json
{
  "extraKnownMarketplaces": {
    "marketing": {
      "source": { "source": "github", "repo": "coreyhaines31/marketingskills" }
    }
  },
  "enabledPlugins": {
    "marketing-skills@marketing": true
  }
}
```

これで「そのプロジェクトを開いているときだけ有効」になる。他のプロジェクトの context は食わない。

開発中に単発で試したいだけなら `--plugin-dir` フラグ（単一セッションのみテスト）も選択肢。

## 選択肢の比較

| 方法 | context消費 | 更新追従 | 独立性 | 備考 |
|------|-------------|----------|--------|------|
| `~/.claude/skills/` に配置 | 全セッションで常時 | 手動 `git pull` | 低 | ラク、だが重い |
| プロジェクトの `.claude/skills/` に直置き | そのプロジェクトのみ | 手動 | 高 | Git追跡自由、一番シンプル |
| `enabledPlugins` 経由 | 有効化したプロジェクトのみ | `/plugin update` | 中 | 本体はグローバル領域にあるが、enable しない限り context に載らない |
| `SKILL.md` をプロンプトに直貼り | 貼ったときだけ | - | 最高 | 永続登録ゼロ、最ミニマル |

## 推奨

- **軽さ・独立性重視** → プロジェクトの `.claude/skills/` に必要なスキルだけコピー
- **更新追従が欲しい** → `extraKnownMarketplaces` + `enabledPlugins` をプロジェクトの `.claude/settings.json` に書く
