## Cotoami Remake とは

**Cotoami（言編み・言網）** は、個人やチームの知識を「概念ベースの知識ベース」へと育てていくための**ノートアプリ**です。クロスプラットフォームのスタンドアロンアプリで、オフライン動作する点が特徴です。元々あった Web アプリ版 [Cotoami](https://github.com/cotoami/cotoami) を、デスクトップアプリとしてゼロから作り直したものが「Remake」です。

技術スタックは、バックエンド/コアが **Rust**（Tauri ベース、SQLite を使用）、UI が **Scala.js + React + The Elm Architecture**。リポジトリ内訳は Rust 164ファイル・Scala 207ファイルで、`cotoami_node`（DB・ノード・プラグインAPI）、`cotoami_desktop`（Tauri + UI）、`cotoami_plugins`（ChatGPT 等）に分かれています。現行バージョンは v0.11.0。

中心となる概念は2つです。**Coto**（こと＝情報の最小単位の投稿）と、それらを繋ぐリンク **Ito**（いと＝糸）。Coto を Ito で繋ぐことで、孤立したメモが相互につながった知識の網になります。さらに **Cotonoma**（言の間＝Coto の集まる空間）という特殊な Coto があり、フォルダ／カテゴリのようでもあり、チャットルームのようにも機能します。

## 「ストック型とフロー型の両方を扱える」について

その指摘は的確で、README にも明確に書かれています。各 Cotonoma は2種類の情報のコンテナになります。

- **フロー（flow）**: タイムラインに流れていく Coto の集まり。チャットやマイクロブログのように気軽に投稿していくもの。
- **ストック（stock）**: 重要な Coto を「ピン留め」して Stock エリアに置き、Pin（Cotonoma 起点の Ito）で階層的に整理したもの。

つまり、Twitter的に流れていく「フロー型」のメモと、Evernote/アウトライナー的に整理して残す「ストック型」のメモを、**1つのアプリ内で同じ Coto を使って両立**できる設計になっています。「気軽に投げる」と「整理して残す」が分断されないのがポイントです。

さらにこのアプリ独自の **Knowledge Growth Cycle（知識成長サイクル）** があり、フローに溜まった Coto のうち中心的になったものを Cotonoma へ「昇格（promote）」させることで、知識が育つにつれて新しいカテゴリが有機的に生まれていく — という、フロー→ストックの動的な橋渡しの仕組みも備えています。

その他の特徴として、オフライン内蔵の世界地図（アプリ容量約500MBの大半がこの地図データ）に Coto を地理情報付きで紐付けられること、SQLite ファイルを WebSocket で繋いで複数データベースをグラフとして統合する「データベースネットワーキング」、WebAssembly ベースのプラグインシステム（ChatGPT 連携など）があります。データは全てローカルの標準 SQLite に保存されプライバシー重視、という思想です。
