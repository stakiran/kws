# 日記プロトコル — 見出され駆動を禁じ、概念提出に純化する
**事実**：書く力は問題ではない。XXXX・Quiet Life・ASDM・10万字/2ヶ月＝完遂実績あり。失敗するのは「ブログ＝見出されエンジン」の枠組みであって、書くことではない。

## 惹きつける動機（過去ページに反復）
1. 見出されたい（discovery）……外部依存・減衰する
2. ニューロダイバシティを広めたい……外部依存・減衰する
3. **つくった概念を提出・展示したい……自己完結・最も安定 ★これだけが本物の駆動**

## 潰す要因（反復）
- だるさ：コセンス変換、整形、画像コピペ
- 無力ループ：「どうせ見い出されない」
- Waveパターン（yr 2025明記）：公開前は高揚→反応なし→燃え尽き→飽き
- フィードバックの矛盾：欲しがるが、来ると腹立つ／来るまでそわそわ

**診断**：動機を「反応という来ない報酬」に縛った瞬間に死ぬ。反応非依存の概念ダンプ（TIC, astad）は生き残る。これが分岐点。

## あるべき姿＝プロトコル
1. 日記本体はコセンスに置く（摩擦ゼロ・変換税ゼロ）
2. 公開面（note / GitHub README）は「概念の純粋ダンプ」に限定。反応を期待しない・保守負担を負わない
3. 更新せず「続報は別記事」方式にする
4. tier3（公開日記）は趣味としてのみ可。見出され狙いに格上げしない
5. 1日1h以上を要求する形式、見られる前提の形式は採用しない

## 禁止事項（毎回これで死んでいる）
- 「これでバズるか／見出されるか」を企画の軸にすること
- 反応の有無を継続判断の基準にすること
- 媒体を増やして両立・連携で手間を膨らませること

**判定基準**：着手前に問う「反応ゼロでも、これを出したいか？」 YESなら出す。NOなら出さない。それだけ。
