# 井戸端における共同空間と個人スタイルのすれ違いについての観察

Cosense（旧 Scrapbox）コミュニティ「井戸端」を読んでいて感じた違和感を、議論を通じて整理した記録。特定個人の問題というよりは、共同 Wiki における共通して起こりうる構造的なすれ違いとして残しておく。

## 出発点となった違和感

直近のページ一覧（新着）を眺めていて、特定の傾向を持つページが大量に並んでいる時期があった。具体的には:

- 「◯◯三人衆」「◯◯四天王」「◯◯三大巨匠」のようなパターン名のシリーズ
- 人物ページなのに中身がほぼ Wikipedia への外部リンク一行だけ
- 一度のセッションで 10 ページ以上の単位で投下されている

これらが新着欄を埋めることで、井戸端本来の「誰かの今日の雑談」「思いつき」「主観の混じった文章」が押し流されているように感じた。

辞書的・名鑑的なページは、書き手の文脈や主観を持たないことが多く、リンク機能による「思わぬ繋がり」が生まれにくい。Cosense のリンクの面白さは、自分の文脈と他人の文脈が予期せず交差するところにあるはずで、辞書的ページはその効果を薄める方向に働く。

## 共同 project と個人 project の非対称

このパターンを生んでいる人は、自身の個人 project を別途持っており、そちらでは数万ページ規模の百科事典的な蓄積を作っている。個人 project でそうしたスタイルを取ることは、本人の自由であり、労作として尊敬に値する。

問題はそのスタイルが井戸端という共同空間にそのまま持ち込まれていること。同じ行為でも、

- 個人 project では: 自分が有用なら問題ない
- 共同 project では: 新着一覧という共有のフロントページに対する占有として作用する

という非対称がある。この論点は井戸端内の過去の議論でも、別の参加者によって既に指摘されている。

## 過去の議論で見えるパターン

井戸端には関連する議論ページが複数ある。スタブページの許容範囲を問うページ、リンクだけのページ作成への異議申し立てページ、URL だけのページの是非を問うページ、他者の文章を改変することの是非を問うページなど。これらを読み比べると、批判される側の応答にいくつか共通したパターンがある。

**1. フレームの引き戻し**
共同空間での振る舞いについての批判が、「異なる方針で運用している個人 project への干渉」と読み替えられる。論点が共有空間ではなく個人領域の話にずらされる。

**2. 検証要求からの後退**
「いつごろの話か」「具体例を見せてほしい」といった要求に対して、「かなり昔」「思考が中断された」「あまり面白い話ではない」「済んだことなので蒸し返したくない」といった形で応答が回避される。主張は維持されるが根拠は提示されない。

**3. カテゴリーによる遮断**
意見表明は「お気持ち」、行動による修正は「編集合戦」、というように、相手が取り得る選択肢それぞれに別カテゴリーを当てて遮断する。結果として批判側が動ける範囲が狭まる。

**4. 形式的対称性の利用**
「私はこれを気持ち悪いと思うので、できれば作らないでいただきたい」という構文を、自分が望む方向にも自分が拒否する方向にも適用できる形にしておく。形式的には対等に見えるが、実態としては一方の希望は通り、もう一方は通らない、という非対称が残ることがある。

**5. 検証可能性の要求を人格的に受け取る**
具体性を求められると「私のことばは偽物だと言われている」のように、内容の検証要求を人格への攻撃に翻訳して詩的に応答する。応答する義務から自分を解放するフレーミング。

## 共同編集性の拒絶を示す記号

Cosense は設計思想として「他者がブラケットで繋いでくる」「割り込みで思考が広がる」ことを前提にしたツール。この共同編集性は Wiki やノートツールとの差別化点。

ところが、自分の日記ページを記号的に囲って「立入禁止」を示す装飾を施す、共有日付ページを使わず自分専用の private な日記運用に分離する、過去に「割り込み禁止」を示すアイコンを作っていた、といった行動が並ぶことがある。これらは個人の領域宣言として表現されている。

個別に見ればただの個人的趣向だが、同じ人が同時に共有空間に対しては大量出力を続けている場合、「自分の領域は守られるべき / 共有空間には自分の出力を流し込んでいい」という非対称が浮かび上がる。

## 読み手の不可視性

井戸端の議論には構造的な問題がある。書き手は議論ページに登場するが、**読み手は黙って去る**。違和感を持っても、それを言語化して書き込むコストを払う読み手は少数派で、結果として議論は「アクティブな書き手」同士の応酬になりがち。

この非対称により、辞書的ページ大量投下の問題も「書き手のスタイル論」としてしか扱われず、「読み手のフロントページ体験」という視点が議論に乗りにくい。

批判される側が時折「読む側の意見も知りたい」と書くことがあるが、いざ読み手が声を上げても、それが上記のフレーム（個人 project への干渉、お気持ち表明、編集合戦の予兆）のいずれかに分類されて処理される傾向があり、読み手のコメントは届きにくい構造になっている。

## どう対処するか

書き込んで指摘するという選択肢は、以下の理由で割に合わないと判断した。

- 既に長期にわたって繰り返されているパターンで、論理的に筋を通す参加者が何度も挑んでも事態は動いていない。
- 「立入禁止」記号を自身のページに既に配置している時点で、外部からの介入が「侵入」として認識される下地ができている。
- 介入が逆に占有的振る舞いを強化するトリガーになりうる。

抽象論として別ページを立てる選択肢もある（個人を名指しせず、論点だけを置く）。これは特定個人を批判する形では書きにくい他の参加者が乗りやすくなる効果が期待できるが、こちらも消耗する可能性は残る。

現状の判断としては:

- 当面は書かずにスルーする。
- さらに状況が悪化した場合は、自分が井戸端から距離を取るか、それが受け入れがたい場合のみ動く。
- (ここは伏せる)
- 思っていることと、やることは別の問題として扱う。

## 残しておきたい一般的な論点

特定の個人の話を離れて、共同 Cosense project 一般に成り立つ論点として:

- **個人 project と共同 project は同じスタイルで運用すべきではない**。後者には「他の参加者の表現を押し流さない」という配慮が必要になる。
- **新着一覧（フロントページ）は希少資源**である。Cosense のインフラ的にはページ数は無制限だが、参加者の注意の総量は有限で、新着欄に何が並んでいるかが「コミュニティの今日の空気」を決める。
- **読み手は黙って去る**ため、書き手だけの議論で運用方針を決めると、読み手の体験が反映されない方向に偏りうる。
- **共同編集性を前提とするツールにおいて、領域宣言は構造的に強い主張**である。「立入禁止」記号を共有空間に持ち込むことは、ツールの設計思想に対する個別の拒絶として機能する。
- **形式的対称性は実質的な非対称を覆い隠すことがある**。「私もあなたも嫌なことを表明していい」という対称ルールは、表明の効果が両側で同じであることを保証しない。

## 結び

井戸端は読むだけでも面白いコミュニティであり、特定の傾向への違和感を理由に全体を切り捨てる必要はない。距離感を調整しつつ付き合う。書きたいことは別の場所にも置ける。

このメモは、自分の中の整理のために残す。
