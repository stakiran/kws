# q
技法と道具の違い

# a
**技法**と**道具**の違いは、ひとことで言うと次の通りです。

- **技法**：やり方・手順・技能（「どうやるか」）
- **道具**：作業に使う物・器具（「何を使うか」）

## 具体例
- 料理  
  - 技法：炒める、煮る、低温調理、包丁さばき  
  - 道具：フライパン、鍋、包丁、温度計
- 絵・工作  
  - 技法：ぼかし、グラデーション、版画の摺り方  
  - 道具：筆、絵の具、パレット、彫刻刀
- スポーツ  
  - 技法：シュートのフォーム、フェイント、投げ方  
  - 道具：ボール、ラケット、スパイク

## 関係性（補足）
- 道具は**技法を実行するための手段**になりやすい  
- 同じ技法でも道具が違うと結果が変わる（例：筆の種類で描き味が変わる）  
- 道具が同じでも技法が違うと結果が変わる（例：同じ包丁でも切り方で仕上がりが変わる）

どの分野（美術、料理、研究、プログラミング等）での「技法と道具」か、文脈があればそれに合わせてもう少し具体的に説明できます。

# q
以下の DWMY Review は技法ですかね？プレーンテキストのテンプレをつくっているが、テンプレは道具なの？技法なの？

# DWMY Review
👉️[English Version](index_EN.md)

## 概要
DWMY Review は個人の、個人による、個人のための振り返りシステムです。

DWMY とは Daily, Weekly, Monthly, Yearly の略です。

## 考え方
- 1: 毎日メモを取ります（デイリーノート）
- 2: 一日一回、デイリーノートを使って振り返りをします（日次レビュー）
- 3: 週に一回、デイリーノート 7 本をインプットにして振り返りをします（週次レビュー）
- 4: 月に一回、週次レビュー結果 4-5 本をインプットにして振り返りをします（月次レビュー）
- 5: 年に一回、月次レビュー結果 12 本をインプットにして振り返りをします（年次レビュー）

このように積み重ねていくことで、現実的なボリュームで振り返りを重ねていくことができます。

もし DWMY がなかったとしたら、週次レビューのときに 7 日分の詳細を思い返したり、月次レビューのときに 30 日分の詳細を調べ直したりする羽目になります。おそらく不可能でしょう。一方、DWMY を心がけていると、週次のときは日次の結果 7 本を見ればいいだけですし、月次のときも週次の結果 4-5 本を見ればいいだけです。現実的に振り返りができるのです。

## ====

## 使う手段
以下のいずれかが良いでしょう。

- テキストエディタや IDE を使って、プレーンテキストで書く
- Cosense、その他軽量でページを自在につくれるウィキやノートで書く
    - 🐰生成AIに与えたいならプレーンテキストでコピペできるもの、あるいはエクスポートできるものが良いです。私は Cosense をおすすめしています

## 何を書くか
Daily

- 日々のタスク、アイデア、作業メモ、日記など何でも
- 日次レビューの内容
- ファイル名の例: `2025-12-31.md`

Weekly

- この週が含む日のデイリーノート（へのリンク）
- 週次レビューの内容
- ファイル名の例: `2025-12-w4.md`, `2025-12-28-wr.md`

Monthly

- この月が含む週の、週次レビュー結果へのリンク
- 月次レビューの内容
- ファイル名の例: `2025-12.md`, `2025-12-31-mr.md`

Yearly

- この年が含む月の、月次レビュー結果へのリンク
- 年次レビューの内容
- ファイル名の例: `2025.md`, `2025-12-31-yr.md`

## レビューとして何を行うか
テンプレートをつくった上で、機械的に埋めるようにすると楽に書けます。以下例:

- YWT: やったこと、わかったこと、次にやること
- KPT: Keep（続けたいこと）、Problem（問題）、Try（次に試すこと）
- 4Y: やること、やったこと、良かったこと、良くなかったこと

また生成 AI に書いてもらう前提で整えるのもアリです。以下例:

- [coppai-sta](https://github.com/stakiran/coppai-sta)
    - これは🐰が使っているプロンプトです（`%cb%` に内容が入ります）
    - 👉️[週次レビュー用](https://github.com/stakiran/coppai-sta/blob/master/wr_for_stary.md)
    - 👉️[月次レビュー用](https://github.com/stakiran/coppai-sta/blob/master/mr_for_stary.md)

# a

DWMY Review は、全体としては **「技法（方法論／運用手順）」**に分類するのがいちばん自然です。  
理由は「何を・いつ・どの粒度で・何を入力にして振り返るか」という**手順と設計**が中核だからです。

## DWMY Review は技法？道具？
- **DWMY Review（Daily→Weekly→Monthly→Yearlyで積み上げる運用）**  
  → **技法**（振り返りのやり方、プロセス設計、レビューの運用ルール）

- その中で挙げている「使う手段」（テキストエディタ、Cosense 等）  
  → **道具**（実行環境）

- 「Markdownでこう書く」「ファイル名をこうする」「リンクで束ねる」  
  → これは境界があり、だいたいは **道具寄りの技法**です  
  - Markdown/プレーンテキスト自体はフォーマット＝道具寄り  
  - 「リンクで入力を束ねて次のレビューのインプットにする」は運用＝技法寄り

## 「プレーンテキストのテンプレ」は道具？技法？
結論から言うと **テンプレは基本的に道具**です。

- **テンプレ（雛形のファイル、見出し、質問項目の枠）**  
  → **道具**（記入を助ける器具、チェックリスト、フォーム）
- **テンプレを“機械的に埋める”という運用**  
  → **技法**（やり方のルール）

さらに言うと、テンプレは「技法を毎回ブレずに実行するための治具（じぐ）」で、**技法の再現性を上げるための道具**、という位置づけがしっくりきます。

## あなたの文章を分解すると（例）
- 「Dailyを書き、日次レビューし、それを週次・月次・年次へ積み上げる」→ **技法**
- 「KPT / YWT / 4Y を使う」→ **技法**（振り返りのフレームワーク）
- 「KPT を書くための Markdown テンプレ」→ **道具**
- 「生成AI用プロンプト」→ **道具**（ただし“どう使うか”の手順は技法）

## まとめ
- **DWMY Review = 技法（運用方法）**
- **テンプレ = 道具（雛形）**
- **テンプレを毎回同じ手順で埋める運用 = 技法**

もし「テンプレも技法と呼びたい」なら、テンプレ単体ではなく **“テンプレ駆動でレビューする”**という一体の仕組みとして呼ぶと整理がきれいです。
