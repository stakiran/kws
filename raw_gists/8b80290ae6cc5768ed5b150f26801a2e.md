# vscode-scb と tipsboard の違い

あなたの記事「Obsidian(Markdown) よりも快適に書く: vscode-scb で Cosense 記法」に、tipsboard 作者の kyu999 さんが「似た動機で作っている」とコメント。両者はどこが似ていて、どこが違うのか。

## 結論

「**ローカルの VS Code で Cosense（Scrapbox）の書き心地を再現する**」という動機とコア機能はほぼ同じ。だから勧められた。
ただし、

- **vscode-scb** = 書き殴る作業場（PTW）に振り切った**極小ツール**
- **tipsboard** = ローカル版 Obsidian 寄りの**多機能 PKM**

という性格の違いがある。あなたの「Markdown も多機能さも要らない」という思想とは一部ぶつかる。

## 似ているところ

| vscode-scb の主張 | tipsboard 側の対応 |
|---|---|
| ローカルで Cosense の書き心地がほしい | VS Code 内の Markdown ウィキ／PKM。サインイン不要・独自DB不要 |
| `[リンク]` のシングルブラケットだけでリンク | `[Project Ideas]` 形式の wiki リンク（`[Label](path)` の Markdown リンクとは別扱い） |
| ページ先が存在しなくてもアクセスできる | suggested new links / isolated-note notice など未接続ノート前提の作り |
| SaaS は遅い・有償・許可が降りない → ローカル | No sign-in / No proprietary DB。Git・Dropbox・Syncthing と連携、ロックインなし |
| 生成 AI（Claude Code）と連携したい | プレーンファイル保存で AI から直接読み書き可。2-hop 発見も標準装備 |
| `.scb` 以外を汚さない | 管理情報は `.tipsboard/` に隔離。ノート本体は素の Markdown のまま |

特に **「`[ ]` シングルブラケット + ローカルプレーンファイル + 未存在ページOK + AI連携」** の4点が「Cosense の感じ」の核心そのもの。kyu999 さんが「ハマるかも」と言った根拠はここ。

## 違うところ（注意点）

あなたが記事で「Obsidian も Markdown も**物足りぬ／オーバースペック**」と書いた、その方向に tipsboard は寄っている。

- **フォーマット**: vscode-scb は `.scb`（箇条書きベース、1-indent 1-space）。tipsboard は**素の Markdown（.md）**。あなたが「ライトな用途にはオーバースペック」と言ったその Markdown。
- **機能量**: tipsboard はバックリンク／2-hop／タグ／**セマンティック検索**／**Kanban**／**Canvas（Mermaid 問題構造図）**／添付ファイル管理…と**かなり多機能**。「Obsidian は多機能ゆえ小回りが利かない」というあなたの評に近い性格。
- **UI**: vscode-scb は普通のエディタ上で完結。tipsboard は専用パネル（Webview）でカード/グリッド表示する、ややビジュアル寄りの体験。

## まとめ

- 動機とリンク哲学（Cosense ライク・ローカル・AI連携）は**双子のように似ている** → だから勧められた。
- ただし vscode-scb が「**書き殴る作業場（PTW）に振り切った極小ツール**」なのに対し、tipsboard は「**ローカル版 Obsidian 寄りの多機能 PKM**」。
- あなたの「Markdown と多機能さはむしろ要らない」という思想とは**一部ぶつかる**。
- 試すなら「箇条書きの軽さ」より「バックリンク／Canvas／セマンティック検索が刺さるか」で評価するとよい。
