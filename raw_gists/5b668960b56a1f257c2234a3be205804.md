# 編集画面
- 入力するもの
  - タイトル
  - ファイル
    - ファイル名
    - インデントモード（tab or n-space）
    - ラップの有無
- テキストエリア
  - **Write/Preview選択方式じゃなくて WYSIWYG 方式**
  - WYG はファイル名（に書かれたファイルタイプ）に従う。Markdown にしたいなら `HOGE.md` にする

# 複数ファイルの挙動
サンプル: https://gist.github.com/stakiran/efe25551c8db99658283ec36b986c14f

- 閲覧時: 1ページ中に全ファイルを表示＆全内容表示（省略表示はない）
- 編集時: 全ファイルのテキストエリアを出す＆全内容表示（省略表示はない）
- **ファイルの並びはファイル名昇順ソート固定**
- 見出しに付くアンカー名は **全ファイル中で共通**
  - [例:「ちょっと長いファイルだよ」](https://gist.github.com/stakiran/efe25551c8db99658283ec36b986c14f#%E3%81%A1%E3%82%87%E3%81%A3%E3%81%A8%E9%95%B7%E3%81%84%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%81%A0%E3%82%88)
  - Expect: `(FILENAME)-(ANCHOR-NAME)`
  - Actual: `(ANCHOR-NAME)`
  - つまり file1.md と file2.md がどちらも `#About` と書いてた場合、両方とも `#about` になる
    - `<a name="user-content-about" href="#about">`
    - repo の GFM では `#about` と `#about-1` になるのに……
