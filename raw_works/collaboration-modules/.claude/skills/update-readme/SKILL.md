# README.md 更新スキル

README.md のモジュール一覧（Conceptware テーブルと Contextware テーブル）を、modules/ 配下の実ファイルから再生成して更新せよ。

## 手順

### 1. 現状の収集

- `modules/concept/` と `modules/context/` 配下の全 .md ファイルを列挙する
- 各ファイルの冒頭を読み、**モジュール名**（先頭の `# 見出し`）と**概要**（最初の説明文から1行要約）を取得する
- ポインタファイル（内容が `see: ファイル名` のみ）は、概要を「→ 参照先モジュール名参照」の形式にする

### 2. テーブルの生成

Conceptware と Contextware それぞれについて、以下の形式のMarkdownテーブルを生成する:

```
| モジュール名 | 概要 |
|---|---|
| モジュールA | 概要テキスト |
```

- 既存の README.md のテーブル内容・順序・表現をできるだけ踏襲する
- 新規モジュールがあればテーブル末尾に追加する
- 削除されたモジュールがあればテーブルから除去する
- 概要の表現は既存のものを優先し、変更が不要なら変えない

### 3. カウントの更新

- README.md 内の「全 XX モジュール」の数字を実際の合計に更新する
- 各セクション見出しの「（XX モジュール）」の数字も実際の数に更新する

### 4. 適用

- README.md を Edit ツールで更新する（Write による全面書き換えではなく差分編集を使う）
- 変更があった箇所をユーザーに報告する
- 変更がなければ「最新の状態です」と報告する

## 注意事項

- README.md の構造（冒頭の説明文、使い方の例、AI エージェントの利用セクション等）は変更しない
- モジュール一覧テーブルとカウント数字のみを更新対象とする
- コミットは行わない（ユーザーに任せる）
