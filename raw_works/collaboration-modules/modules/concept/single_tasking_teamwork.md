# Single-Tasking TeamWork
**Single-Tasking TeamWork（STTW）** とは、シングルタスク駆動的なチームワークを指す。

STTW では以下を遵守する:

- 行動単位は「タスク」とする
    - タスクとは「1人が 3 日以内に完了できる or 完了できなくても中止できる」程度のもの
- メンバーにはシングルタスク制約を課す
    - 同時に一つのタスクにのみ取り組める
    - 別のタスクに取り組みたい場合、今持っているタスクは完了、中止、中断しなければならない
- タスクは SSoT（信頼できる唯一の情報源）として扱う
    - メンバー全員が読み書きできるツールを確保し、タスクは必ずここに書き込むこと
    - タスクには単一の URL（Permalink）が存在し、URL を提示するだけでそのタスクについて言及できること
    - トピック指向を心がけること。タスクも含めて1つの話題を1ページで書き、他の話題が混ざる場合は別ページに移す
    - タスクには統一的な ID を振ること。これはタスクを統一的かつ手間なく指せるようにするためである。連番でもランダムな文字列でも何でも構わないが、口頭で言及しやすい程度に短くする
    - タスクとタスクを関連付けられる（リンクできる）ツールが望ましい。たとえばタスク #12 と #35 を踏まえてタスク #67 を開始する場合、#67 の文脈として #12 と #35 へのリンクを入れておくことで、誰でも文脈を追えるようになる
- タスクの担当者は必ず一人とし、意思決定権も担当者が担う
- タスクには複数人の助言者を設定できる。助言者が設定された場合、担当者は必ず助言者に一度は助言を請わねばならない
    - ティール組織の助言プロセス
- 完了、中止、中断はうまく使い分ける
    - 完了とはタスクを正常に終えたことを意味し、れっきとした成果があるはずだ
    - 中止とはタスクを終えられなかったことを意味し、終えられなかった理由や事情があるはずだ
    - 中断とは完了も中止もしていないが、いったん継続できないので一時的に止めるという意味である。あとで再開することができるし、再開は別の担当者でも良い。再開するのに必要なだけの情報は揃えておかねばならず、別の担当者が再開できない場合は中断とは言えない。この場合、再開できるようになるまで対応してもらうか、中止にするかを催促する
- 完了、中止、中断のタイミングで必ずタスク（のページ）を更新し、いつ誰が見ても理解できるようにしておかねばならない
    - これを行わない者には厳しく催促するべきだし、これを行えない者は行えるようになるまで教育するべきである。STTW はタスクの SSoT 化が必須であり、タスクは各担当者が残すものである。残す行動をしない者は害悪となる

STTW はシングルタスクという制約を端的に示したものであり、上記以外の細かい運用は規定しない。したがって、STTW のみでプロジェクトを回すというよりも、既存の運用に STTW を追加する形で使う。

STTW は共通言語でもあり、メンバー全員がタスクに基づいたコミュニケーション、合意形成、意思決定を行うこととなる。たとえば作業や検討のサイズは1人で3日以内に終了できるサイズにまで落とさなければならないし、不明点が多くとも3日以内でいったん区切って、別タスクを作り直さねばならない。

必要ならタスクのフォーマットを定めてもいいが、多くの場合、創造性を損ねるし、一部メンバーの主義思想ともバッティングする。おそらく単一フォーマットでの統一は非現実的だろう。複数のフォーマットを許容するか、最低限何を書いてほしいかを定めて最低限書いてもらうのが良い。生成 AI でビューを整える方法もある。
