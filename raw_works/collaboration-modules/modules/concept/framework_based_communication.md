# Framework Based Communication
**Framework Based Communication** とは、コミュニケーション手法とその使い方を定義した上で、それに従ってコミュニケーションを行うもの。

構成要素は次のとおり:

- Framework/
    - Steps/
        - step1
        - step2
        - ...
    - Templates/
        - template1
        - template2
        - ...

まずコミュニケーションの最小単位をステップ（一往復分のやりとり）と呼び、これを Template Based Communication で行う。次に、どのステップをどう組み合わせて流すかを設計する。この一連の設計をフレームワークと呼ぶ。

## 体験
Framework Based Communication では、対象者全員がフレームワークに厳格に従う。雑談、議論、意思決定まであらゆるコミュニケーションをカバーできるが、ステップが Template Based Communication であるため、全面的にテンプレートの提供とその記入をもって進行する体験となる。

また、会議のように同期的な機会として実施する必要はなく、非同期的にも実施できるし、繰り返しになるが雑談でも使える。無論、テンプレートを「読み書き」するため、口頭よりもリアルタイム性は落ちやすい。

折衷案として、テンプレートに書いて記入するのではなく、テンプレートに從って口頭で回答することで記入とみなす（口頭記入）案もあるが、テンプレート以外の発話を制限するのがおそらく難しい。軍隊レベルの規律がなければ実現はできないだろう。
