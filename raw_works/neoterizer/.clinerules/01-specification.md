# Neoterizerプロジェクト

# 概要
- Cosense プロジェクト https://github.com/stakiran/neoterizer を Markdown 化します

# 処理

## 入力
- neoterizer.json を読み込みます

## 前処理
- マスク化します
    - neoterizer.json を読み込み、user.id と user.email の内容をマスク化します
    - マスク化とは、同じ文字数の `X` に置換することです
- マスク化した後、neoterizer_masked.json として保存します

## 本処理
- neoterizer_masked.json を読み込み、1-ページ 1-Markdown ファイルに変換します。
- 保存先は docs/ ディレクトリです
- 各ファイルのファイル名は「ページタイトル」を「Writable Name」に変換したものを採用します
- 各ページは「Cosense 記法」で書かれており、これを Markdown に変換します。次のように変換します:
    - インデントなしの行は `# 大見出し` で記述します
    - インデントが1つ以上ある行は、`- リスト記法` で記述します
        - インデントが1つの場合は ` - リスト`、インデントが2つの場合は `    - サブリスト`、インデントが3つの場合は `        - サブサブリスト` のように、4スペース単位で深くしてください
    - 内部リンクは `[ページ名](ページ名のWritableName.md)` に変換します
    - 外部リンクは `🌎️[テキスト](URL)` に変換します
    - Cosenseプロジェクトへのリンク `🌎️[プロジェクト名](https://scrapbox.io/プロジェクト名)` に変換します
    - 画像は HTML の img タグに変換します
    - アイコン記法も HTML の img タグに変換しますが、各行に文字として挿入できるくらい小さく表示します。また
        - またページ名へのリンクもつけてください。たとえばアイコン記法が `[sta.icon]` であるなら、変換後の Markdown は `<a href="sta.md">imgタグ</a>` のようになります
    - コード記法は、Markdown のコードブロックに変換してください。使用するシンタックスハイライトも指定してください。たとえば `code:filename.md` であるなら、シンタックスハイライト指定も `md` となります
    - テーブル記法は、Markdown のテーブル記法に変換してください

### 各ページの json は次のように書かれています
以下はページ `sta` の例です。

```json
{
    "title":"sta",
    "created":1750325710,
    "updated":1750547794,
    "id":"6853d9cc45c90571907b8b03",
    "views":30,
    "lines":[
        "sta",
        "[https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b]",
        "このサイトの管理人です。",
        "ブログ: [/stao]",
        "",
        "",
        "個人用設定:","
        code:script.js  ",
        " import '/api/code/sta/sta/script.js'",
        "code:style.css  ",
        " @import \"/api/code/sta/sta/style.css\";",
        ""
    ]
},
```

- ページタイトルは title キーです
- ページ内容は lines キーです。1行1要素で記述されています。Cosense 記法で書かれています
- ページ作成日時は created キーです

### Writable Name
Windows のファイル名として書き込める名前を意味します。

具体的には、ページタイトルに `\` `/` `:` `*` `?` `"` `<` `>` `|` ` ` の文字が含まれている場合に、`_` に置換します。

### Cosense 記法
内容は行指向であり、1行1要素の文字列配列で保持されます。

1-インデント 1-ホワイトスペースで構成します。ホワイトスペースは半角スペース、全角スペース、タブがありえます。以下に例を示します。

```
これは行です
これは行です

箇条書き
 レベル1です
 レベル1です
  レベル2です
   レベル3です
 レベル1です

以下はいずれもレベル1です
 半角スペースによるレベル1です
　全角スペースによるレベル1です
	タブによるレベル1です

以下はいずれもレベル2です
 ここはレベル1
  ここからレベル2
　　これは全角+全角のレベル2
 　これは半角+全角のレベル2
		これはタブ+タブのレベル2
	 これはタブ+半角のレベル2
```

空行で区切られたn行の塊をチャンクと呼びます。以下に例を示します。6個のチャンクがあります。

```
これはチャンク1です
あああ
 いいい
 ううう

これはチャンク2です
あああ
いいい
ううう
ここもまだチャンク2です


チャンク3です
ここもチャンク3です

これはチャンク4です

チャンク5
チャンク5
チャンク5

チャンク6
 チャンク6
チャンク6
```

文字装飾は次のとおりです。Cosense では `[]` で囲む形の文法が多いです。

- 内部リンクは `[ページ名]` と書きます
- 外部リンクは `[テキスト URL]` または `[URL テキスト]` と書きます
- Cosenseプロジェクトへのリンクは `[/プロジェクト名]` と書きます
- 文字の拡大は `[* 拡大された文字]` と書きます。`*`の数が多いほど文字は大きくなります
    - `[* 少し大きい文字]`
    - `[** やや大きい文字]`
    - `[*** 結構大きい文字]`
- 画像は `[https://gyazo.com/XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX]` と書きます
    - X にはランダムは文字列が入ります
    - 実際のファイルは `https://gyazo.com/XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX.png` として存在します
- アイコン記法は `[xxxx.icon]` と書きます
    - これはページ名 `xxxx` に記載された、一番上の画像を小さく表示する記法です
    - 典型的には `xxxx` は人名であり、プロフィール画像が掲載されています

表（テーブル）は次のように書かれます。以下は2行3列のテーブルです。テーブル記法と呼びます。

```
table:テーブル名
	1行目の1列目	1行目の2列目	1行目の3列目
	2行目の1列目	2行目の2列目	2行目の3列目
ここはテーブルの内容ではありません
```

つまりテーブルとは、

- `table:テーブル名` という行で始まり、
- 次行から1インデントして、タブ区切りでセルを書いていき、
- インデントがなくなる行に来るまでテーブルが続きます

また、テーブルのセルは空値もありえます。

```
table:これは2行2列目が空値のテーブルです
	1行目の1列目	1行目の2列目	1行目の3列目
	2行目の1列目		2行目の3列目
```

コードブロックは次のように書かれます。コード記法と呼びます。

```
code:コード名.拡張子
 1インデント下げてコードを書く
 1インデント下げてコードを書く
 1インデント下げてコードを書く
 ...
```

つまりコードとは、

- `code:コード名.拡張子` または `code:拡張子` で始まり、
- 次行から1インデントしてコードの内容を書いていき、
- インデントがなくなる行に来るまでコードが続きます

コードのシンタックスハイライトは `拡張子` の部分で判断します。

なお、テーブル記法とコード記法にはインデントがつくことがあります。

```
インデントなしの行
インデントなしの行
 インデント1の行
 code:このコードはインデント1の中で書かれたものです
  コード
  コード
 インデント1の行
  インデント2の行
  table:このテーブルはインデント2の中で書かれたものです 
  	1行目の1列目	1行目の2列目	1行目の3列目
  	2行目の1列目		2行目の3列目
```

# 処理の実装
- Python で実装します
- ファイル名は convert.py です
- 実行は `python convert.py` です
    - 余計なコマンドラインオプションは与えず、必要なパラメーターはスクリプト内部で与えます
