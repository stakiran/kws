# Closed Complex.

# 背景
- 現代は[VUCARD](VUCARD.md)であり、オープンな視座が要求される
    - 内輪で情報を握るのではなく、多様や持続への投資として自らを開いておくということ
    - 例: 短期的で明確なインセンティブがなくとも情報を公開する、なるべく皆に見えるところでやりとりする
- 一方でオープンのあり方はマジョリティにとってハードルが高い
    - 1. 人は想定された範囲にのみに情報を届けたがる
        - [📖PLURALITY 対立を創造に変える、協働テクノロジーと民主主義の未来]ではニッセンバウムの「文脈の完全性」を挙げています。また、プライバシーという言葉を「想定された範囲内で完結する」のように定義し直しています
    - 2. オープンは近年インターネットによって開拓された新しいあり方であり、インターネットやテクノロジーに長けた者でないとその感覚が養われない
    - 3. SNSの盛り上がりにより炎上を始めとするリスクが起きやすくなり、クローズド化が再燃している
        - コミュニティをDiscordサーバーでゲーテッドにするのは良い例でしょう
- これではいつまで経ってもオープンが進まないため、進めるための概念が必要と考えました<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
    - マザコンやファザコンと同じノリを採用しました。いつまでも親（クローズド）に依存してないで、自立しましょう（オープンになりましょう）とのメッセージ性を込めています

# 日本文化との適合性
    - > 日本の職場は 「ウチ／ソト」「根回し」「リング制度」 など、事前に限定的な関係内で合意形成を図る慣行が強い。したがってクロコンというラベルは 現状を可視化するレンズとしては有効。ただし「集団調和」を尊重する価値観ゆえ、いきなり全面オープンを押しつけると反発されやすい。
- 弱点のカバー
    - スティグマ化リスク → 指標化して漸進的にする
        - >スティグマ化リスク ― 「クロコン＝悪」と烙印を押されると、当事者が防衛的になり逆効果	ラベリングから指標へ：個人やチームを“クロコン／オープン”で二分するのではなく、「情報公開率」「公開チャネルでの議論比率」など定量 KPIに変換し、漸進的改善を促す（例：四半期ごとに +10 % を目標）。
    - 短期イベント
        - >「クロコン・デトックス week」: 1 週間だけ DM を禁止し、必ず公開チャンネルで議論→効果を測定し改善点を共有。短期イベント形式にすることで抵抗感を下げ、オープン文化のベネフィットを体験学習させる。
- 総評
    - >「クロコン」は既存研究（文脈的完全性・知識隠蔽・ダークソーシャル等）の枠組みに照らして 概念的には妥当であり、実務でも Slack や InnerSource といった成功事例が後押しする。ただし 価値判断が強いためスティグマ化しやすい点が弱みで、定量 KPI 化や体験イベント導入で漸進的に文化をシフトさせるアプローチが望ましい。
