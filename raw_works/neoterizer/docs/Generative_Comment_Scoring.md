# Generative Comment Scoring
# 生成AIを用いて、コメントを共感ベースでスコアリングするというゲーミフィケーションの方式。

# 背景
- フィードバックでもパブリックコメントでも何でもいいが、コメントがほしいとする
- コメントを出してもらうにはインセンティブが必要
    - しかし単純な報酬だと脆弱性を突かれてしまい、システムや他プレーヤーが搾取される
- 上手いインセンティブの設計方法が欲しい

# アプローチ
- 1: 早くコメントした人がえらい
- 2: 共感が集まったコメント（を書いた人）がえらい
- この二点をベースに、コメントをスコア付けすると良い
- 生成AIを使えば現実的にできるのではないか

# 概要
- 1-トピックに対してN-コメントがくると想定する
- 以下2つのデータをつくる
    - 1: コメントを時系列で保存した生データ
    - 2: 1からスコア付けを行ったデータ
    - 以下はものすごく単純な例
<img src="https://gyazo.com/7796f6a37d7434bfe983fa3fa81fc903.png" />

- スコア付けのアルゴリズムについて(イメージ):
    - 動的にカテゴリをつくり、分類していく
        - ここが最大の特徴であり、新しいコメントが投稿されたときに、その場でスコア付けの処理が可能となる。多くの生成AIソリューションと違ってバッチ処理ではなくリアルタイム処理が可能である<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
    - 各カテゴリに放り込まれた個数 = 共感の数であり、スコアである
    - 配当はスコアに基づいて行われる
        - 各カテゴリの、早くコメントした者への配分が多くなる
