# AISI
# 生成AIでSIを変革していくという考え方。

# 背景
- 無論、様々な考え方が存在するが、たしかなビジョンとアプローチを示さねば活動しづらいです。そこで参考となる枠組みを与えることで、SIerのみなさまにインスピレーションを与えます<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>

# 概要
- 記事より抜粋します<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
- 1行で
    - >AISI では「SI という強制約活動を AI でカバーするには？」に取り組みます。
- 3点 + 手法
    - >1: 情報を引き出す
        - >人間から引き出すための「チャット・ストーミング」
        - >[DCPC](DCPC.md) から引き出すための「収集」
    - >2: 情報を LLM に渡す
        - >Markdown 前提で立ち回る「Markize」
        - >埋め込みベクトル化を継続的に行う「継続的エンベディング」
    - >3: 情報を使って実装や構築を行う
        - >AI を用いた省力化や AI への委譲を行う「[デリファメーション(Delifirmation)](デリファメーション(Delifirmation).md)」
