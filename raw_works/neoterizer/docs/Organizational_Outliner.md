# Organizational Outliner
# 階層的な組織での運用を前提とした[アウトライナー]、またはそのようなあり方。
# 組織的なアウトライナー。

# 背景
- 現代であっても組織は階層的に動く（動かざるをえない）ことが多い
    - [カスケードダウン]と呼ばれる
    - 特に計画や目標を組織的に統一的に定める際には重宝される
- しかしそのためのツールは原始的であり、[PWEP](PWEP.md)でファイルとして管理するか、[HCM]（人事人材管理/Human Capital Management）に入力させるかが現状である
    - 要するに全面的に手作業的であるため、認知コストから時間まで多くのコストがかかっています。もちろん人手なので正確性も出づらく、結局評価者らの政治となってしまって公平性も失われがちです<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
- 端的に読み書きする手段があればいいと考え、アウトライナーに目をつけました<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>

# 概要
- [🙏組織単位](🙏組織単位.md)としてアウトライナーをサポートする、と考えればよい
    - TeamsのチームやSlackのワークスペースなど、組織単位ごとに「場」をつくるのは鉄板である
    - 「階層的な整合性を持たせねばならない情報」も、この仲間に加える
