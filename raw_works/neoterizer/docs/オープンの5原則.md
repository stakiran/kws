# オープンの5原則
# いつでも、誰でも、自由に、かんたんに、かつ利活用しやすい状態で見れる状態にすること。
# オープンのニュアンスは様々だが、この原則ではこのように定める。

# 略称はAFREA(アフレア)
- Anytime, Anyone, Free, Easy, Readableのアナグラム
    - 検討ログ: [za_KhSJzr](za_KhSJzr.md)

# 概要
- いつでも: 24/7見れる
- 誰でも: 社内関係者だけでなく全世界誰もが見れる
- 自由に: 承認や登録なしですぐに見れる
- かんたんに: ブラウザから読める程度にかんたんに見れること
    - 文書ファイルではなくWebページが望ましい
    - →[ページズム(Pagism)](ページズム(Pagism).md)
- 利活用しやすい状態で: 機械可読可能であること
    - かんたんにとの相反するので、事実上複数の提供形式を用意する必要があります<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>

# 関連
- FAIR原則
🌎️[The FAIR Data Principles – FORCE11](https://force11.org/info/the-fair-data-principles/)
    - 科学データの公開原則で、Findable, Accessible, Interoperable, Re-usableの略
    - AFREAとは関係がありません。FAIRは科学データの利活用性にフォーカスしており、オープンの観点はあまりありませんたとえばAFREAとは違って閲覧に条件をつけることができます<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
