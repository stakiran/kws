# 思想
- 人と人が直接つながるんじゃなくて、用語と用語をつなげる
    - 用語であれば、社会性がなくてもやりやすい
    - 情報密度が濃くて非同期的なやりとりを進めやすい
- 技術的に「全ユーザーの全用語集とつながる」は無理だし、構造的にも良くないので分散させる
    - [Fediverse]におけるインスタンス、Discordでいうサーバーに相当するものを採用する
    - ネットワークと名付けました。サーバーやインスタンスは言葉が技術的すぎますし、コミュニティやグループやギルドは社会的すぎて思想に合いません。最初はLibraryや-eum（～～館）などの路線で攻めていましたが、本だとウィキの構造をイメージしづらいのでやめました。次に広場やパークといった路線で攻めましたが、所属感が薄く、一時的な来訪のイメージが強いのでやめました。ここで書庫(Archive)を思いつき、倉庫・貯蔵のニュアンスで攻めましたが、そこまで静的かつ死蔵的ではありませんので不採用です――と、ひととおり粘りまして、原始的にウィキ構造のニュアンスを持たせられればそれでいいと妥協して、ネットワークにしました<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
- 要はウィキなので何でも書けるが、用語を書いてほしい
    - 同じ用語があったときにその違いや規模を把握したい、見える化したい、整理したい
    - より大胆に言えば、実は[知的生産](知的生産.md)のSNSにしたいのです<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
- 個人主義を損なわない
    - 用語集は各個人の持ち物であり、それを持ち寄る構造にする
    - これにより用語集の管理やメンテナンスは、引き続き持ち主が主体的に進められる
    - コミュニケーションではなくコラボレーションがしたいのです。各個人が自分のペースで深くつくったものを持ち寄ることで化学反応を生み出したいのです。しかし個人のペースは乱したくないのです<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>

# 細かい設計
- ページからページにはリンクを張れます
    - リンクのあり方は4種類あります:
        - 1: 自分内、同じ用語集内
        - 2: 自分内、違う用語集間
        - 3: ユーザー間リンク。自分の用語集W1と相手の用語集W2間
        - 4: ネットワーク内リンク。ネットワークN内の、自分の用語集W1と相手の用語集W2間
        - 個人的には種類が多すぎるのでシンプルにしたいですね。同一空間(1)と異なる空間(2,3,4)くらいで二分できる気はします<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
- ページには以下の種類があります
    - 用語。フリーフォーマットです。SNGなので用語を書くのが推奨ですが、もちろん何を書いてもいい
    - リンク集。他の用語へのリンクを並べたものです。リンクを並べる以外のことはできません
    - エイリアス。他の用語へのリンクを一つだけ張ったものです。これ以外の操作はできません
    - 典型的には用語を使えばいいですが、リンク集も使えます。また古い用語や同義語などはエイリアスを使って整理します
- ページは[ダブルノート(Double Notes)](ダブルノート(Double_Notes).md)が良いでしょう
    - つまりPublicとPrivateの2枚があって、普段表示されるのはPublicだけです
    - Publicには載せない下書きやゴミなどはPrivate側に書きます。Privateは自分だけが見ることができます
    - UIとしては、たとえばページの上部にタブが二つあって、publicとprivateを切り替えます
