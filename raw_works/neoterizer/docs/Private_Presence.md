# 例2
- グループスケジュール上のプライベート予定の内容を具体的にする
    - Before: 「Private」
    - After: 「🐶さんと～～の件で打ち合わせ」
    - その人のスケジュールを見ることで、その人が何してるかを知ることができます<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
- [APP-DM（Auto Private Presence For DM）]
        - 名前は仮です。良い名前が浮かびません<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
    - いつ、誰とDMしたのかが自動でリストされるという仕組み
    - これにより記録されたものをDMログという
        - 🐶さんのDMログ → 🐶さんがいつ誰とDMしたかが全部見える
        - グループスケジュールと同様、社員全員が見ることができます<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
        - この概念は知的生産物であり、まだ具体的な製品はないはずです。DMログはセンシティブな可能性が高いですから、もう少し細かいアクセス権限が必要かもしれませんね<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
    - 実は透明性という観点では、Private ChannelよりもDMの可視化こそが重要だったりします。ここまで踏み込めるほど透明性に注力した企業は見たことがありません（単に私が知らないだけでしょうが）。なぜかというと、純粋に従業員や経営者に要求されるリテラシーが高いからです。要は「いつ誰と喋ったか」をフルオープンにするわけですが、現代人はまだここに忌避感を持っています<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
