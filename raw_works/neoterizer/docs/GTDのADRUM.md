# GTDのADRUM
# GTDで行う操作を5つで述べたもの。
# 読み方は「アドラム」

# 開発秘話
- そろそろ[GTD]を生成AIベースで扱えるでしょう、というわけで当方でも検討しています。そのためにはGTDの概念を言語化して捉えねばなりません。ADRUMもその過程で生まれました
- なお、ここでいうGTDは、[📖GTDを噛み砕く](📖GTDを噛み砕く.md)をベースにしています
    - ※造語が多いです。たとえばinboxやprojectなどの「情報を入れる先」を「箱」と表現したりします

# 概要
- それぞれ以下の略。※なおGTDのフローとして自然な順で書く
    - Dump 脳から外に出す
    - Add 外に出したものを箱に入れる
    - Move 箱から箱に移す
    - Remove 箱から捨てる
    - Unpack 項目から外に出す
- 用語について:
    - 箱: 情報を入れる先。インボックス、プロジェクト、連絡先、ネクストアクションなど
    - 項目: 箱に入れる情報一つ分
    - つまり、ここでは（GTDを噛み砕くでは）GTDを箱――もっと言えば項目からなるリストの集まりとその運用ととらえています<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
