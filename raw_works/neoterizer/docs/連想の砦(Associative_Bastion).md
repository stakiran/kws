# 連想の砦(Associative Bastion)
# 交流、議論やアイデア出しにおける連想（発散）を行うためのやり方のこと。

# 概要
- マジョリティは連想の砦が一つしかない
    - ≒対面で同期的に会話すること
    - 例: [会って喋るほうが効率的に交流できる時代 - しゅみは人間の分析です https://non117.com/entry/2025/01/04/214232]
        - >対して対面での交流では会話のボールを投げながら連想的に思い出したことをぽいぽい話すことができる。
- 連想の砦は一つではなく、増やすことができる
    - 例1: [拘束的非同期](拘束的非同期.md)
        - 一緒にいるが喋らず、かわりに読み書きでやりとりする
        - これにより非同期の利便性と対面の非言語情報を両取りできます<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
    - 例2: [QWINCS](QWINCS.md)
        - リアルタイムに同時編集ができるノートやボードに情報を書き、それを膨らませていくことでも連想は行えます<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
    - 例3: [フォームニケーション(Formunication)](フォームニケーション(Formunication).md)
        - n人がフォームに自由に意見を投稿し、かつ投稿結果も全部見れるようにすることで、思い出したことを何でも書き込めます<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>

# 開発秘話
- ITエンジニアなど優れた人であっても、連想を要する高密度な議論や交流は対面で行うことがわかってきました。これは好ましくありません。働き方の多様性がちっとも向上しません。何が足りないのかを調べていて、連想のメンタルモデルが偏っていることを突き止めました。これを連想の砦と表現してみました。砦が一つしか知られていないのです。他にも増やさねばならないのです
- 仕事で、限定的であればできる人も多いですが、カジュアルな交流になるとお手上げです。これは「カジュアルに行えるほどのメンタルモデルやスキルがない」とも言えます。鍛えてもらうのは酷ですが、道具でまかなうことはできます。道具化は[知的生産](知的生産.md)の範疇外ですが、今回私がこの概念をつくったことにより、皆さんはそういう道具をつくるという発想を持てるようになりました
