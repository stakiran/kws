# 未読アイコン
# 未読メンバーを示すアイコンのことで、既読になると消える。

# 概要
- [既読エンジニアリング](既読エンジニアリング.md)の一種で、既読の問題点を軽減するために、既読ではなく未読の表示を扱う
    - 具体的には以下を軽減する:
        - 未読者フォローの手間。未読アイコンは未読者をそのまま表示しているのでフォローもしやすい。
        - [認知的安全性(Cognitive Safety)](認知的安全性(Cognitive_Safety).md)の低下。未読アイコンだと未読者が見えるため、次の状態（全員読まれるまであとどれくらいかかりそうか）を読みやすい。
        - 未読者への圧の少なさ。未読アイコンは未読者をある種晒す形になるので、サボタージュを抑止しやすい。
- BFAF
    - Before: 既読した人（のリアクションや既読ステータスなど）が表示される
    - After: 未読の人が表示される。読むと消える
