# バックリマインダー(Back Reminder)
# 離席から復帰したときに発動するリマインダーのこと。

# 背景
- 離席から復帰した場合、何らかの未反映が脳内で溜まっていることが多い
    - 例: 思いついたアイデアがある（ので残したい）、思い出したタスクがある（ので仕込みたい）、買い物をした（ので家計簿に書きたい）
- これを回収するためにはリマインダーを使えばいいと考えました。タイミングとしては、離席から復帰したときに想起させればいいでしょう。これを離席から戻って来るということでBackの名前を借りて、Back Reminderとしました<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>

# 実装
- 考え方
    - 大前提として「作業環境」がある
        - AFK(Away From Keyboard)という言葉がある。キーボードやデスクや部屋は典型例だろう
    - 作業環境に戻ってきたときにリマインドさせるといい、と考える
    - その作業環境において「戻ってきたこと」をどうやって検出するか、を考える
- 例
    - 安価のセンサで自分の動きを検出
    - マウスやキーボードの動作をソフトウェア的に検出
    - チャットツールなどのステータスの変化を検出
    - [- 復帰ボタンをつくってそれを押す]
        - ただこの場合は「ボタンを押すとリマインドされる」というおかしな挙動になります。個人的にこれはバックリマインダーとは認めたくありません。これは本質的には「席に戻ったら反映しなさい！」のようなメッセージを机に貼り付けていることと同義であり、そのうち慣れて、スルーするようになって形骸化します。自発性は排除して、仕組みのほうから気づかせてくれることが必須です<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
- リマインド内容の洗練
    - おそらく時間帯ごとに「何をリマインドさせればいいか」は決まってくると思いますので、それを反映します。日中の散歩だとアイデアを思いついている可能性が高い、夕方のちょっとした外出は買い物をしている可能性が高い、など。もしそのような傾向がない場合は諦めてください<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
