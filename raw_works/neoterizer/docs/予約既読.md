# 予約既読
# 既読した旨の表示を、指定日時にまとめて実施すること。

# BFAF
- Before: 既読すると即座に「🐶さんが既読」のような表示される
- After: たとえばその日の既読は23:59に表示される
    - なので8:03に見ようが、17:45に見ようが、23:02に見ようが同じこと

# 概要
- [既読エンジニアリング](既読エンジニアリング.md)の一種
- 予約既読により、[認知的安全性(Cognitive Safety)](認知的安全性(Cognitive_Safety).md)を高めることができる
    - 1. 既読はCSを下げる。「読んだ」という情報を即座に相手に与えるし、それを受けた相手がいつ行動してくるかがわからないため
    - 2. もちろん、既読をつける行動自体をこちらで制御するのは、その分の認知資源を使っており、やはりCSは良くない
    - 3. 3つ目のやり方として、予約既読を使うと、1と2の間を取れる。今すぐには既読をつけないため1を回避できるし、しかし後から自動でつけはするので2の消耗もない
