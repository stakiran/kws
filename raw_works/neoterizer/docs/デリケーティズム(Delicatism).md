# デリケーティズム(Delicatism)
# その領域に関して繊細な者こそがその領域を広げるのだという考え方。

# 名言
# >いつだって、繊細な者が拓くのだ。

# 概要
- 繊細とは:
    - 外発的なインセンティブがなくとも、主体的に（あるいは自動的に）、常人が理解できないほど細かい部分にまでこだわること
    - 捉える解像度が細かいと言ってもいいと思います<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>

# 拓きたいと願うならば、あなたが繊細でいる領域を特定して、そこで勝負するといいでしょう。もちろんすぐにキャリアやファンや仲間の獲得に通じるとは限りませんが<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>

# 例
- 既存のプログラミング言語の書き方や安全性に納得できない、繊細な者が新しいプログラミング言語を生む
    - Python, Rubyなど軽量言語 → Golang, Rustなどの現代的に書きやすいが堅牢な言語
- 既存の働き方の偏りや愚かさに納得できない、繊細な私が新しい働き方を生む<a href="sta.md"><img src="https://gyazo.com/a0a22d2fc5cf4fb2525db091fb66594b.png" alt="sta" width="16"/></a>
