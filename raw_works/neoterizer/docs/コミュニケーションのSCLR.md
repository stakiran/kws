# コミュニケーションのSCLR
# [🙏コミュニケーション](🙏コミュニケーション.md)がもたらす性質を4つで整理したもの。
# 読み方は「スクル」。

# 概要
- 同期性 Synchronicity
    - 同期か非同期か
    - リアルタイムにやりとりするなら同期、それ以外は非同期
    - 対面だから同期、チャットだから非同期とは限らない
        - 対面でも非同期は可能 → [ラグカッション(Lagcussion)](ラグカッション(Lagcussion).md)
        - チャットでもお互い待ちながらリアルタイムに出し合うなら同期になる
- 拘束性 Constraintivity
    - 時間、場所、話題を拘束しているかどうか
    - 例
        - 対面で会議 → 時間と場所と話題を拘束
        - リモートで会議 → 時間と話題を拘束
        - [拘束的非同期](拘束的非同期.md) → 時間と場所を拘束
    - 参考: [コミュニケーションにおける「3つの拘束」 | monolithic https://stakiran.github.io/monolithic/3_communication_restraint.html]
- 言語性 Linguisticity
    - 言語か非言語か
- 往復性 Reciprocality
    - どれだけ往復するか（やりとりするか）
    - 以下が存在する:
        - 1往復
        - n往復
        - 時間内は無限往復
        - 無制限に無限往復

# 意義
- コミュニケーションの改善をSCLRからアプローチできるようになる:
    - 同期性を変えよう（同期なのがネックだから非同期にしよう）、往復が無いから最低限2往復させよう、など具体的なアプローチが可能となる
