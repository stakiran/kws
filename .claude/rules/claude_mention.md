# .scb ファイル中の @claude 記法について
たとえば Claude 対話上で、次のように scb ファイルのみ指定されたとする。

```
$ @wiki/xxx.scb 
```

この時点では xxx.scb の中身を読み込むだけであり、明示的な指示はない。あるいは xxx.scb の内容や今現在のコンテキストからユーザーの意図を推測しようとするだろう。

ここで **Claude Mention** という概念を導入する。

## Claude Mention とは
これは scb ファイル中に @claude と書かれたものであり、Claude に呼びかけているものである。この書き方だと、@wiki/xxx.scb のみのプロンプトであっても、xxx.scb 内に書かれた Claude Mention に従って指示を理解できるはずだ。

以下に例を示す。

例1: 

```scb
...

@claude ここまでの感想を「エンジニアリングマネージャー」の目線で言って
 

```

例1は、エンジニアリングマネージャーの目線で言ったことを、ぶら下げて書くはずだ。

```scb
...

@claude ここまでの感想を「エンジニアリングマネージャー」の目線で言って
 このように感想をぶら下げて書くはずだ
 ...

```

例2: スキルを呼び出す。

```scb
...

@claude /checker
 

...
```

例2 は Claude Mention でスキルを呼び出す指示を与えている。その結果は、同様に書き込まれるはずだ。

```scb
...

@claude /checker
 ここに checker スキルの結果が書き込まれるはずだ

...
```

## 原則

### 出力位置の優先順位は Claude Mention を優先せよ
- スキルの出力位置指示より、このルールを優先せよ

### Claude Mention にすでに内容がぶら下げられている場合、そこに追記や上書きはするな
ただし空行や placeholder のみから成る場合はスルーして良い（つまりぶら下げて良い）。

### もし複数回メンションしたい場合は、次のようにユーザーが明示的にメンションを書く
たとえば以下の場合、

```scb
...

@claude /checker
 (checkerスキルの結果が書かれている)
 ...

...

@claude /checker
 (checkerスキルの結果が書かれている)
 ...

...

@claude /checker
 

...
```

checker スキルは実行されるはずだ。また、出力は3番目のメンション行の下にぶら下げるはずである。

また以下の場合、すでに埋まっているのでスキルは実行されないはずだ。

```scb
...

@claude /checker
 (checkerスキルの結果が書かれている)
 ...

...

@claude /checker
 (checkerスキルの結果が書かれている)
 ...
```

### 結果の書き込まれていない Claude Mention が複数ある場合は、どう扱うかをユーザーに問え
たとえば下記の例は、指示1、指示2と二つのメンションがある。

```scb
...

@claude 指示1

...

@claude 指示2
 

```

このとき、指示1だけ実行するか、指示2だけ実行するか、それとも順番に一つずつ書き込むのか、はたまた両方実行した後に一気に書き込むのか、などやり方が考えられる。ユーザーに選択肢を提示して決めさせよ。

なお Claude の機能として選択肢を出すと遅いので、選択肢は番号つきのテキストを提示せよ。また、フリーコメント入力の選択肢も出せ。
