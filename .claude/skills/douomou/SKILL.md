# douomou — どう思う？

ユーザーが書いた .scb ページに対して、Claude の「学習モデル由来の率直な感想」を返すスキル。

## 入力

- 引数として .scb ファイルのパスが渡される
- 引数が省略された場合は、直近の会話で扱っていた .scb ファイルを対象とする。曖昧なら 1 度だけ確認する

## 手順

1. 対象 .scb ファイルを Read する
2. 本文中の `[リンク名]` をすべて抽出する
3. 各リンクについて **1-hop だけ** 対応する .scb ファイルを Read する
   - ファイル名は CLAUDE.md の `fixInvalidFilename` ルールに従って解決する（`\ / : * ? " > < |` とスペースを `_` に置換）
   - 通常は対象ファイルと同じディレクトリ（多くは `wiki/`）を見る
   - 該当ファイルが存在しないリンクはスキップ（リンク切れは黙って無視）
4. 読んだ内容について、**Claude 自身の学習知識のみ** から感想を生成する
   - リポジトリ内の他ファイル（他の .scb、`raw_works/`、`raw_gists/`、git log など）は参照しない
   - Web 検索もしない
   - 「世間一般・学習データ上で知られていること」と「ユーザーが書いた内容」とを突き合わせ、共感・違和感・補足・別視点・関連概念などを出す
5. 出力フォーマット（下記）に従って、**対象 .scb ファイルの末尾に追記（append）** する
   - 会話には「追記しました」程度の短い報告のみ返す（感想本体を会話で繰り返さない）
   - 既存内容は一切編集しない。末尾に追加するだけ
   - 追記前に対象ファイル末尾が改行で終わっているか確認し、必要なら 1 行空ける

## 出力フォーマット

対象 .scb ファイルの末尾に追記する。scb 記法の 1-space 1-indent 箇条書きで書き、AI コメントであることを示すため **🤖** を付ける（ユーザーアイコン 🐰 と区別する）。

追記ブロックの先頭には、感想セクションであることがわかる見出し行を置く（例: `🤖 どう思う？`）。複数回 append しても区別がつくよう、見出し行に簡単な日付を添えてもよい。

書き方は次のいずれでもよい:

- 行頭に 🤖 を置く
- 行末に 🤖 を置き、その配下にインデントでぶら下げる（CLAUDE.md の 🐰 運用と同じ）
- 必要なら `[ページ名]` 形式で関連ページにリンクしてよい（存在しないリンクでもよい。scb ではリンクから新規ページを切り出せる）
- 感想は率直に。賛同だけでなく違和感・反例・別解釈も書く
- 元ページの要約は不要。ユーザーは元ページを読んでいる

## やらないこと

- 1-hop を超えてリンクを辿る
- 対象ファイルの既存内容を編集する（追記のみ。書き換え・削除はしない）
- リンク先 .scb ファイルへの追記（追記先は引数で指定された対象ファイルのみ）
- リポジトリ内の他資料を踏まえた整理（このスキルはあくまで「外部知識からの感想」専用）
