# checker — 概念ページの 5 観点点検

ユーザーが書いた .scb の概念ページに対し、**背景・系譜・道具・反証・経験**の 5 観点で点検し、結果を末尾に追記するスキル。

## スタンス

- 評価はスコア化しない。読み返しのための目安として使う
- 5 観点すべてを満たす必要はない（満たさない観点があってもよい）
- ✅ の基準は**厳しめ**に判定する

## 入力

- 引数として .scb ファイルのパスが渡される
- 引数が省略された場合、IDE で開いている .scb ファイルを対象としてよいかユーザーに 1 度だけ確認する

## 5 観点と ✅ 条件

各観点には「効用」が紐づく。✅ 条件は厳しめに判定する。

- **背景** が設定されている → 共感できる
  - ✅ 条件: 困っている具体的な状況が複数（3 つ程度）、それぞれ十分な記述で書かれている
- **系譜** に接続されている → 検証できる
  - ✅ 条件: 先行概念との比較が明示的に書かれている
- **道具** が整備されている → 使用できる
  - ✅ 条件: 読者が即実行できる手順、または実装へのリンクがある
- **反証** が提示されている → 判別できる
  - ✅ 条件: 機能しないケース、または棄却条件が書かれている
- **経験** に接続されている → 実感できる
  - ✅ 条件: 具体的な体験エピソードが本文に含まれている

## 手順

1. 対象 .scb ファイルを Read する
2. 5 観点それぞれについて ✅ 条件を満たすか判定する
3. 各観点ごとに、判断理由を一行で書く
4. 出力フォーマット（下記）に従って、**対象 .scb ファイルの末尾に追記（append）** する
   - 会話には「追記しました」程度の短い報告のみ返す
   - 既存内容は一切編集しない。末尾に追加するだけ
   - 追記前に対象ファイル末尾が改行で終わっているか確認し、必要なら 1 行空ける
   - 既に「チェッカー」ブロックが存在しても、置き換えず新しいブロックを追記する

## 出力フォーマット

scb 記法の 1-space 1-indent 箇条書きで書く。AI コメントであることを示すため **🤖** を付ける（ユーザーアイコン 🐰 と区別する）。

5 観点すべてを記載し、✅ 条件を満たした観点のみ ✅ を付ける。満たさない観点は ✅ を付けず、理由だけを書く。

```
🤖チェッカー
 背景 ✅ <理由を一行>
 系譜 <理由を一行>
 道具 ✅ <理由を一行>
 反証 <理由を一行>
 経験 ✅ <理由を一行>
```

## 書き方の指針

- 判断理由は一行で簡潔に。長くなりそうなら、何が足りないか（または何が満たされているか）を 1 点に絞る
- ✅ を付ける際も、なぜ満たしていると判断したかを一行で添える
- 元ページの要約や全体評価コメントは不要。ユーザーは元ページを読んでいる

## やらないこと

- リポジトリ内の他ファイル（他の .scb、`raw_works/`、`raw_gists/`、git log など）を参照する
- Web 検索する
- スコア化（合計点や 5 段階評価など）
- ✅ 以外の判定マーク（❌、△ など）を出力する
- 5 観点のうち一部だけを書く（必ず 5 観点すべて記載する）
- 対象ファイルの既存内容を編集する（追記のみ。書き換え・削除はしない）
- 既存「チェッカー」ブロックの削除や書き換え
- タイムスタンプを付ける（履歴管理は意図しない）
